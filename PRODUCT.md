# Product

## Register

product

## Users

主要用户是希望成为 AI Agent 应用工程师、Agent 平台工程师或技术负责人的学习者。他们可能具备不同程度的 Python 和后端基础，但共同目标不是浏览概念，而是围绕同一个 OpsPilot 企业工单 Agent，完成可运行实现、失败实验、评测证据和架构答辩。

辅助用户包括带教学员的工程师、课程维护者，以及需要检查学习证据是否达到岗位要求的面试官或评审者。

## Product Purpose

本产品把分散的 Agent 知识组织成一条可执行的工程学习路线。每个阶段都要求学习者理解责任边界，复现真实故障，实现最小修复，完成正常、失败和对抗验证，并保存可复现证据。

成功不是“读完资料”，而是学习者能够独立解释取舍、运行实验、定位失败、提交证据，并把 OpsPilot 从严格契约逐步演进到具备 Runtime、Tool/Policy、RAG、Durable Execution、Memory、Eval、Observability、Security、Multi-agent 和 Release 治理的系统。

## Brand Personality

严谨、清醒、务实。语气像一位愿意说明原理和事故边界的资深工程师，直接但不居高临下。界面应让学习者感到路线明确、证据可信、下一步可执行。

## Anti-references

- SaaS 营销落地页：不使用大幅口号、装饰性数据或产品宣传替代真实任务。
- 泛化 AI 工具面板：不使用紫色渐变、玻璃拟态、发光装饰和没有信息作用的卡片墙。
- 文档堆放站：不把大量链接等同于课程，不让学习者自己猜顺序和完成标准。
- 聊天式假教学：不只给最终答案，不用主观“看起来会了”替代测试、评测和证据。
- 伪工业级演示：不把前端规则模拟、研究论文结果或 S0-S2 教学基线声称为完整生产系统。

## Design Principles

1. **一条项目主线**：所有知识都回到 OpsPilot 的输入、输出、责任层和故障证据。
2. **失败先于结论**：先复现事故，再解释机制和实现修复。
3. **证据决定进度**：完成状态由前置检查、自测、实验和工程证据共同决定。
4. **比较后再选型**：展示每种方法的适用条件、成本、失败模式和替代路径，不追逐单一框架。
5. **边界始终可见**：明确区分教学模拟、研究原型、工程可用模式和生产验证结果。
6. **学习状态可恢复**：进度、错题、实验决策和复述保存在本地，可导出、可重置、不过度收集数据。

## Accessibility & Inclusion

- 目标为 WCAG 2.2 AA，正文和交互状态保持足够对比度。
- 所有核心流程可用键盘完成，焦点状态清晰，不只依赖颜色表达结果。
- 支持 `prefers-reduced-motion`，动画只用于表达状态变化。
- 从 320px 起可用，在 390px 手机宽度和常见桌面宽度下不得出现页面级横向溢出。
- 中文为默认教学语言，代码、标识符、命令和上游项目名保留英文。
- 长表格在窄屏下转换为可扫描的分组内容，避免缩小字体或裁切文字。
