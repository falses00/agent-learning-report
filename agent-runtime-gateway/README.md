# 企业级 Agent 教学项目

项目目标：一步一步构建一个企业级 Agent Runtime/Gateway，让你先掌握设计、评审和验收方法；后续需要时，再把这些设计承接成工程产物。

## 当前阶段

Phase 0：课程总览、路线审核、证据矩阵、教学循环和学习路线。

本阶段不写业务代码，先把“要做什么、为什么这么做、参考哪些靠谱项目、每一步如何学习、如何测评审核”讲清楚。

## 当前学习模式

默认采用 **Design-only**：你不写代码，也不记代码；你要学会的是如何设计、追问、评审和验收一个工业级长线 Agent。

代码实现可以作为后续 **Implementation-later** 阶段由 Codex 承接，但当前每轮学习必须先产出：

- 设计决策卡：为什么这样分层和治理。
- 失败场景卡：如果缺少这一层会怎样失控。
- 架构评审问题：用来挑战方案是否偷懒。
- 用户复述验收：你能不能用自己的话讲清设计。

- [术语表](00-课程总览/术语表.md)：中文显示名、英文标准术语和首次解释。

当前决策：

- 采用 Product Design brief gate + 本地 Open Design PRD。
- PRD 是权威需求，Product Design 插件后续用于视觉探索。
- MVP 只做课程可视化和 mock 数据，不接真实生产系统。
- 每个计划步骤必须先过 PRD 中的深度思考模板、Phase A-F 阶段门禁和学习验证实验。
- 外部资料和 GitHub 项目先进入 LLM-Wiki 知识层，经过 SourceReview、KnowledgeCard 和 KnowledgeVersion 后才可进入课程主线。
- Product Design 可点击原型前，必须先通过 v0.4-pre 冻结包：状态矩阵、mock 数据、逐页验收和学习验证脚本。
- v0.4-pre 的最低门禁：每个页面必须能说清事故、负责层、`source_files`、复述题和低分回流；缺任一项都不能进入原型。
- v0.4 首轮原型范围收敛为 6 页：C 首页、A 全链路、B 事故推演、RAG 诊断器、LLM-Wiki 知识层、证据与门禁页。
- v0.4 静态原型与 Chrome 浏览器 QA 已通过，下一步必须执行学习效果验证，而不是继续扩前端。
- 真正前端工程进入 Implementation-later 后再创建。

## 下一步

进入 Phase 1 前，先确认 MVP demo 场景。

推荐首选：**工单处理 Agent + 只读知识库 + 受控工具调用**。
