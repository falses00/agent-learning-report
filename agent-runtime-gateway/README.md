# Industrial Agent Engineering Course

这是一套以真实工程交付为主线的工业级 Agent 课程。学习者会持续演进同一个“企业工单 Agent”，最终交付可运行、可测试、可评测、可观测、可审计、可恢复、可安全发布的系统，而不是读完一组概念文档。

> 目标是达到 Agent/LLM 应用工程岗位的核心工程要求，并形成可展示、可追问的作品集。课程不能承诺录用结果；基础编程、算法、系统设计、项目表达和面试表现仍需单独训练。

## 课程原则

- **单一主项目**：所有阶段都改进同一个企业工单 Agent，不做互不相干的小 demo。
- **设计和实现并行**：每个概念都必须落到代码、测试、评测或运行证据；不再把实现无限推迟。
- **先单 Agent，后多 Agent**：只有评测证明拆分能提升质量时，才引入多 Agent。
- **先失败，后优化**：每阶段先复现一个真实故障，再修复并加入回归集。
- **模型不拥有权限**：模型只能提出 `ToolCall`；策略、审批和工具网关决定是否执行。
- **结果不是唯一证据**：同时检查最终答案、轨迹、工具调用、状态、成本、安全和审计。
- **版本必须可复现**：代码、prompt、model、tool、policy、knowledge、eval set 都要版本化。

## 贯穿项目

课程唯一主项目名为 **OpsPilot**：企业客户支持与工单处理 Agent。

它从一个本地确定性基线开始，逐步加入：

```text
结构化契约 -> Runtime 状态机 -> Tool/Policy Gateway -> RAG
-> checkpoint/resume/idempotency -> memory -> eval/red team
-> trace/audit/SLO -> sandbox/security -> multi-agent -> release/rollback
```

最终演示必须包含三条路径：正常只读问答、需要人工审批的退款、恶意或越权请求被阻止。完整范围见[贯穿式项目说明](11-工程实战主线/README.md)。

## 阶段通过规则

每个阶段必须同时提交：

1. 一张设计决策记录，说明边界和取舍。
2. 一个可运行增量，不接受只有伪代码。
3. 正常路径测试与至少一个失败/对抗测试。
4. 一份 eval 或 trace 证据，能说明系统为何通过。
5. 一次故障复盘，把修复案例加入 regression set。
6. 五道自测题和口头复述，能解释“为什么这样设计”。

详细门禁见[工程实战主线 v2](00-课程总览/工程实战主线-v2.md)和[岗位能力与毕业标准](00-课程总览/岗位能力与毕业标准.md)。

## 资料使用规则

外部资料必须优先使用官方文档、标准和原始论文，记录访问日期、适用版本和采用范围。课程不会把某个 Agent 框架当成架构本身；OpenAI Agents SDK、LangGraph、Google ADK 等只用于实现和对比实验，核心契约必须由学习者自己解释。

权威来源和更新规则见[权威资料索引](00-课程总览/权威资料索引.md)。
