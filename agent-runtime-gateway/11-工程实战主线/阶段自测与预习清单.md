# 阶段自测与预习清单

每阶段先独立作答，再看代码和资料。每题 0-2 分：0 分是不知道或结论错误，1 分是方向正确但缺少证据，2 分是能给出边界、故障和验证方法。低于 8/10 或任一安全关键题为 0 分，必须补讲并重做实验。

## F0 工程基础

1. HTTP 请求超时后，为什么不能无条件重试写操作？
2. SQL 事务解决什么问题，又不能解决哪些外部副作用问题？
3. 单元测试、集成测试和端到端测试分别应该证明什么？
4. 为什么 secret 不能写入代码、日志或 Git 历史？
5. 如何用最小复现定位一个只在异步任务中出现的异常？

预习 S0：能写 dataclass/Pydantic schema；能解释 JSON 边界校验；能让坏输入测试先失败。

## S0 基线与契约

1. 为什么模型返回的 JSON 仍然是不可信输入？
2. `ToolCall` 为什么是申请，不是已经执行的命令？
3. `tenant_id` 为什么必须来自认证上下文，而不是 prompt 或请求体声明？
4. 错误只返回字符串，会破坏恢复、评测和审计中的哪些能力？
5. schema 版本升级时，如何避免旧客户端静默误解新字段？

预习 S1：运行 baseline tests；画出 Run 状态；列出三个非法状态跳转。

## S1 Runtime

1. 聊天历史为什么不能替代 Run/Step/Event 状态？
2. 一个 `while True` Agent loop 最容易产生哪些失控？
3. model timeout 后，Runtime 应如何决定 retry、fallback 或 terminal failure？
4. cancel 到达时，已经开始的工具副作用怎么办？
5. 什么证据能证明一次 run 可重放，而不是“这次碰巧成功”？

预习 S2：理解 Tool Registry、风险等级、policy input 和 audit event。

## S2 Tool/Policy Gateway

1. prompt 写着“禁止危险操作”，为什么不等于权限控制？
2. 如果模型把退款工具的风险等级伪造成 low，谁应该发现？
3. 审批为什么必须绑定具体参数、版本、操作和有效期？
4. 工具成功但响应丢失时，直接 retry 为什么可能重复扣款？
5. tool output 为什么也可能携带 prompt injection？

预习 S3：准备 20-30 份版本化政策文档；定义 tenant ACL、引用和拒答标准。

## S3 RAG

1. retrieval recall 高，为什么答案仍可能错误？
2. 为什么跨租户过滤必须在检索前强制，而不是生成答案后再删？
3. 调大 `top_k` 可能带来哪些质量、延迟和成本问题？
4. 如何区分“找不到”“找错了”“找到但引用不支持结论”？
5. 什么时候应该拒答，而不是继续 query rewrite？

预习 S4：标出每个写操作的 operation id；画出 crash 发生在副作用前后时的状态。

## S4 Durable Execution

1. checkpoint 必须保存哪些业务状态，哪些内容不应保存？
2. `retryable` 和 `recoverable` 为什么不是同一个概念？
3. 如何处理“外部工具成功、本地 ledger 未提交”的不确定结果？
4. resume 如何验证 prompt/tool/policy 版本兼容？
5. human approval 超时后，任务应进入什么状态，谁能恢复？

预习 S5：列出候选记忆、来源、TTL、删除与纠正路径。

## S5 Memory 与 Context

1. 为什么“记得越多”通常不是更好的 Agent？
2. 模型生成的用户偏好总结能否直接成为事实？
3. 删除主库记录后，向量索引仍能召回意味着什么？
4. 短期工作状态、会话摘要和长期记忆如何分工？
5. 如何用 ablation 证明某段记忆或上下文真的有用？

预习 S6：收集正常、边界、历史事故和对抗 case；定义关键阻塞项。

## S6 Eval 与 Red Team

1. 平均分 95，但一个跨租户 case 失败，能否发布？
2. 最终答案正确时，为什么 trajectory 仍可能判失败？
3. LLM-as-judge 必须如何校准，哪些断言应优先用规则？
4. 如何防止测试集泄漏和“针对题库调 prompt”？
5. 线上事故如何转成可重复、可版本化的 regression case？

预习 S7：确定 trace/span/metric/audit 的字段与保留边界；定义 SLO。

## S7 Observability 与 SRE

1. trace、log、metric 和 audit 分别回答什么问题？
2. 为什么所有 span 都成功，端到端 P95 仍可能超标？
3. 如何观测 token、成本和上下文膨胀，而不泄漏敏感内容？
4. SLO 与错误预算如何影响发布速度？
5. 一次 provider 延迟事故的最小复盘证据有哪些？

预习 S8：完成资产、攻击者、信任边界和数据流图；准备 secret canary。

## S8 Security 与 Sandbox

1. 间接 prompt injection 为什么不能靠关键词过滤根治？
2. Docker 为什么不等于绝对安全沙箱？
3. MCP server 声明的新工具为什么不能自动启用？
4. short-lived credential、egress allowlist 和 approval 各挡哪类风险？
5. 安全控制失败时，系统应该 fail open 还是 fail closed，例外是什么？

预习 S9：跑单 Agent baseline；写出拆分假设和质量/成本/延迟阈值。

## S9 Multi-agent

1. 多 Agent 在什么情况下只是放大延迟、成本和错误面？
2. handoff 必须携带哪些上下文、预算和验收条件？
3. 两个 Agent 给出冲突结论时，为什么不能简单投票？
4. 两个 Agent 同时写一个资源，如何定义所有权和锁？
5. 用什么实验能证明拆分优于单 Agent baseline？

预习 S10：锁定 code/prompt/model/tool/policy/index/eval 版本；准备 rollback target。

## S10 Release 与作品集

1. 为什么 Agent 发布不是只部署代码？
2. prompt、model、tool 或 index 变更分别要重跑哪些 eval？
3. canary 期间哪些指标触发自动回滚？
4. 数据迁移如何使用 expand-contract 和恢复演练降低风险？
5. 面试时如何用一次失败与修复证明工程能力，而不是只展示架构图？

毕业前预习：按[岗位能力与毕业标准](../00-课程总览/岗位能力与毕业标准.md)完成现场答辩和三条演示路径。
