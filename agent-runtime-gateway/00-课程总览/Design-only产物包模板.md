# Design-only 产物包模板

生成日期：2026-07-01  
用途：每节课结束时，把“我听懂了”变成可检查的设计资产。

## 1. 产物包总览

每个阶段只交 6 个小产物：

| 产物 | 用途 |
|---|---|
| 一句话解释 | 防止只记术语 |
| 业务例子 | 把概念落到工单故事 |
| 坏设计卡 | 证明你知道系统怎么失控 |
| 设计卡 | 说明应该加哪层、哪张契约或哪道门禁 |
| 评审问题 | 用来挑战方案是否偷懒 |
| 复述卡 | 检查你能否用自己的话讲清 |

## 2. 一句话解释

```text
Phase __ 解决的是：________。
它防止的核心失控是：________。
```

示例：

```text
Phase 3 解决的是工具调用失控。
它防止的核心失控是 Agent 绕过 Tool Gateway 直接执行高风险工具。
```

## 3. 业务例子

```text
在工单退款故事里，这一阶段对应的动作是：________。
如果没有它，可能发生：________。
```

## 4. 坏设计卡

| 项目 | 填写 |
|---|---|
| 坏设计 |  |
| 为什么看起来省事 |  |
| 会造成什么事故 |  |
| 哪一层应该拦住 |  |
| 应该变成什么评测或审计 |  |

## 5. 设计卡

| 项目 | 填写 |
|---|---|
| 本阶段要加的层/契约/门禁 |  |
| 输入是什么 |  |
| 输出是什么 |  |
| 谁负责判断 allow/deny/error |  |
| 哪些字段必须存在 |  |
| 哪些失败必须阻塞下一阶段 |  |

## 6. 评审问题

每节课至少问 3 个：

```text
1. 如果没有这一层，会出现什么真实事故？
2. 这个方案有没有把权限、审计、评测交给 prompt？
3. 失败后能不能定位到具体层？
```

## 7. 复述卡

```text
这一阶段解决的是 ________。
它在工单故事里对应 ________。
如果缺少它，会发生 ________。
我设计的最小产物是 ________。
我用 ________ 来验收。
我现在仍说不清的是 ________。
```

## 8. 评分

| 分数 | 判断 |
|---|---|
| 0 | 只重复术语 |
| 1 | 能解释术语，但没有业务例子 |
| 2 | 有业务例子，但说不出事故 |
| 3 | 能说出事故和负责层 |
| 4 | 能指出坏设计并提出修正 |
| 5 | 能指挥 Codex 进入实现和验证 |

Design-only 阶段至少 3 分才能继续。
