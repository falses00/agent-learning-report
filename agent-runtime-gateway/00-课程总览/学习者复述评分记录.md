# 学习者复述评分记录

生成日期：2026-07-01  
用途：记录每节课你是否真的理解，而不是只看过文档。

## 1. 使用方法

每节课结束后记录一次：

- 你自己的复述。
- Codex 给你的评分。
- 你卡住的概念。
- 下一轮修复动作。

## 3. 评分规则

| 分数 | 表现 |
|---|---|
| 0 | 只会说术语 |
| 1 | 能说定义，但没有业务场景 |
| 2 | 能套到工单故事，但说不出事故 |
| 3 | 能说出事故、负责层和最小验收 |
| 4 | 能指出坏设计，并设计替代方案 |
| 5 | 能让 Codex 按你的设计实现、验证和复盘 |

## 4. 修复动作库

如果低于 3 分，选一个修复动作：

| 卡住类型 | 修复动作 |
|---|---|
| 名词听不懂 | 让 Codex 用工单故事重讲 |
| 不知道为什么需要 | 让 Codex 给一个坏设计事故 |
| 不知道怎么验收 | 让 Codex 给一个失败样例和阻塞条件 |
| 说不出系统边界 | 让 Codex 画层级关系 |
| 越讲越散 | 本轮只保留一个概念 |

## 5. 进入下一阶段句式

```text
我对 Phase __ 的复述评分是 __ 分。
我能讲清它防什么事故、哪层负责、怎么验收。
我仍不清楚的是 ________，但它不阻塞下一阶段。
可以进入 Phase __。
```
