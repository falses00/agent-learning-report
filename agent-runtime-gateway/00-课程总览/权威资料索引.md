## 7. 技术栈结论

课程采用 **Python 后端/评测主线 + TypeScript 产品界面**。OpenAI Responses API/Agents SDK 可作为主 provider 实现，LangGraph 用于 durable execution 对比，MCP 为工具互操作必修，A2A/ADK 为跨 Agent 选修，OpenTelemetry/OWASP/NIST 贯穿生产治理。

不押注单一框架。框架可以替换，以下能力不可替换：契约、状态、权限、幂等、评测、观测、安全、成本、发布和复盘。

## 8. 更新门禁

进入每阶段前重新检查：

- 官方页面是否仍存在，发布日期和版本是否变化。
- API/SDK 是否 deprecated，迁移截止日期是什么。
- 协议、semantic conventions 和 SDK 是否兼容。
- license、安全公告、数据驻留和 telemetry 默认行为。
- 课程代码锁定的版本能否从干净环境重新安装并通过测试。

资料变更只在有工程影响时修改主线；纯名称或营销更新不应制造课程 churn。
