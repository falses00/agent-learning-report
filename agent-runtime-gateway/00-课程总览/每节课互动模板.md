# 每节课互动模板

生成日期：2026-07-01  
用途：让你每次和 Codex 学 Agent 时都有固定提问方式，避免回答越来越抽象。

## 1. 标准启动提示词

```text
继续 Phase X：{阶段名}。

我现在是半懂不懂，不想写代码，也不想记代码。
请先调用合适的 subagent 增强提示词。
如果涉及最新资料、GitHub、外部文档或方案对比，请调用 researcher。

教学要求：
1. 先用“工单处理 Agent”故事讲，不要直接堆术语。
2. 每个术语都要按“白话解释 -> 为什么存在 -> 少了会出什么事故 -> 工业级怎么验收”来讲。
3. 本轮只教本阶段，不跳到后续阶段。
4. 给我一个坏设计案例。
5. 给我一张设计卡。
6. 给我一张填空式复述卡。
7. 如果我复述不清楚，请停下来纠正，不要进入下一阶段。
```

## 2. 如果我讲得太抽象，你这样打断

```text
先暂停。
我没跟上。
请不要继续扩展名词。
只用工单故事，把刚才这个概念重新讲一遍。
按照：现实动作 -> 系统层 -> 为什么需要 -> 没有会怎样。
```

## 3. 如果你想确认自己懂没懂

```text
请你考我 5 个问题。
不要考定义，考我能不能判断坏设计。
每个问题后等我回答，再指出我哪里理解错了。
```

## 4. 如果你想让 Codex 反思课程是否够好

```text
请站在半懂学习者视角审查刚才这一课：
1. 哪些地方太抽象？
2. 哪些地方缺例子？
3. 哪些地方我可能以为懂了但其实没懂？
4. 下一版教学应该怎么改？
5. 是否能进入下一阶段？
```

## 5. 每节课结束前必须生成

| 产物 | 形式 |
|---|---|
| 一句话总结 | 不超过 30 字 |
| 白话解释 | 不用英文缩写 |
| 专业术语对照 | 术语 + 中文解释 |
| 坏设计案例 | 具体事故 |
| 设计卡 | 字段表或责任表 |
| 复述卡 | 填空题 |
| 停止条件 | 哪些没懂就不能继续 |

## 6. 复述评分标准

你的复述不是越长越好。按这个标准判断：

| 分数 | 表现 |
|---|---|
| 0 | 只重复术语，不知道为什么 |
| 1 | 能解释术语，但说不出事故 |
| 2 | 能说出事故，但不知道系统哪层负责 |
| 3 | 能说出层级、事故和验收 |
| 4 | 能指出坏设计并提出更好设计 |
| 5 | 能让 Codex 按你的设计去实现和验证 |

每阶段至少达到 3 分，再进入下一阶段。

评分结果记录到：

- [学习者复述评分记录](学习者复述评分记录.md)

## 7. 卡住时的三段式修复

```text
第一轮：换成生活化解释。
第二轮：换成坏设计事故。
第三轮：换成填空复述和判断题。
```

三轮后还不懂，就缩小范围：

```text
本节课只保留一个概念，其他全部暂停。
```

## 8. 每节课的最终通过句式

当你能说出下面这段，就算本节课 Design-only 通过：

```text
这一阶段解决的是 ________ 失控。
它在系统里属于 ________ 层。
如果缺少它，会出现 ________ 事故。
我设计的最小产物是 ________。
我用 ________ 来验收它。
现在还不能进入代码阶段，因为 ________。
```
