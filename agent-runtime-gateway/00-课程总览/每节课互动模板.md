# 每节课互动模板 v2

更新日期：2026-07-13

状态：当前有效。替代旧版“默认 Design-only、不写代码”的互动模板。
用途：让每次教学都沿 OpsPilot 工程主线推进，并以可运行实现、失败测试和可复现证据判断是否通过。

## 1. 标准启动提示词

```text
我要学习 OpsPilot 的【阶段/知识点】。我的当前水平是【基础/半懂/可独立编码】，本轮可投入【时间】。

你是工业级 Agent 工程导师。请围绕同一个 OpsPilot 企业工单 Agent，一次性建立本课完整上下文，然后按门禁推进，不要把阅读完成当成掌握。

请输出并执行：
1. 一句话定义知识点，列出 3-6 个关键概念。每个概念按“白话解释 -> 为什么存在 -> 缺少会发生什么事故 -> 如何验收”讲清楚。
2. 指明它在 OpsPilot 中的输入、输出、责任层、信任边界，以及明确不负责什么。
3. 给出一个有来源的公开事故；若没有可靠公开来源，必须标成“合成工程故障场景”，不得冒充真实事故。
4. 先让我复现最小失败，再带我完成最小实现。给出文件位置、命令、预期结果和停止条件，不只给最终代码。
5. 设计正常、失败、对抗三类检查，并说明应留下哪些 test、trace、audit、eval 或报告证据。
6. 给出 5 道设计判断或故障分析题。每题记录首轮答案；答错时指出具体误区、换一个更简单的例子、指定必须重跑的实验和新增的 regression case。
7. 生成一页速记清单、当前阶段证据清单、下一阶段预习核对清单。
8. 映射到岗位能力：面试官会追问什么，作品集必须展示什么，当前证据离毕业标准还差什么。

强制约束：
- 本轮只处理当前阶段，不跳过前置条件。
- 优先引用项目代码、测试、官方文档、标准和原始论文。
- 明确区分：已验证事实、工程经验、合理推断、未来趋势。
- 明确区分：教学模拟、设计练习、可运行教学基线、生产验证。
- 未接入可运行实验的阶段只能标记“理论与设计练习完成”，不得标记“工程通过”。
- 工程通过必须登记：实际命令、正常结果、失败/对抗结果、产物或 commit 引用、边界说明。
- 不得承诺 offer、职级或薪资；只按可复现证据判断岗位准备度。
```

## 2. 教学输出顺序

| 顺序 | 教学环节 | 必须产物 | 常见问题 | 纠正方式 |
|---:|---|---|---|---|
| 1 | 定义与边界 | 一句话定义、输入输出、不负责项 | 堆术语，没有 owner | 回到工单事故，画责任边界 |
| 2 | 故障复现 | 可重复失败、预期与实际 | 只讲结论 | 先写失败断言，再解释机制 |
| 3 | 最小实现 | 文件、命令、最小增量 | 复制大段框架代码 | 缩到一个契约或状态转换 |
| 4 | 三类验证 | 正常、失败、对抗 | 只有 happy path | 从历史事故生成 regression |
| 5 | 证据登记 | 命令、输出、产物引用 | 自报完成 | 要求可重跑路径和结果摘要 |
| 6 | 五题自测 | 首轮分数、误区、重跑实验 | 背定义 | 改成坏设计判断和故障归因 |
| 7 | 收尾与预习 | 速记、岗位映射、下一阶段前置 | 读完就前进 | 未过门禁则回到最小缺口 |

```text
- 更简单的工单例子：

- 修复后要新增的 regression case：
- 请我重新作答的问题：
```

## 4. 五道自测规则

- 至少两道是安全、权限、可靠性或副作用关键题。
- 不考孤立定义，优先考坏设计识别、责任归属和故障恢复。
- 首轮得分低于 8/10，或任一关键题首轮答错，必须完成指定实验后重做整组。
- 重做不是清除历史；证据记录应保留原误区、修复动作和第二轮结果。

## 5. 工程证据格式

每阶段必须登记以下五项，且内容不得包含 secret、真实客户数据或 PII：

1. **边界说明**：负责什么、不负责什么、可信事实来自哪里。
2. **实现与命令**：实际修改的文件、commit 或运行命令。
3. **正常路径**：预期、实际结果和关键输出摘要。
4. **失败路径**：失败/对抗输入、阻断位置、回归结果。
5. **产物引用**：测试报告、trace、audit、eval、ADR 或复盘的本地路径/URL。

只有仓库已提供可运行实验，并且五项证据、课前准备和自测同时通过，才可标记“工程通过”。

## 8. 停止条件

出现任一情况就停止进入下一阶段：

- 只能复述术语，不能说出责任边界和事故。
- 没有真实运行命令或结果摘要，只勾选“已完成”。
- 测试只有 happy path，没有失败或对抗样例。
- 写操作没有 schema、policy、approval 或 idempotency。
- 修改 prompt、model、tool、policy、index 后没有跑回归。
- 阶段工程实验尚未入库，却试图标记工程通过。
- 证据中含 secret、PII、真实客户数据或不可公开信息。
- 多 Agent 没有证明优于单 Agent baseline。
