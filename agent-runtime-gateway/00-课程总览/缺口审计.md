# 缺口审计

> **状态更新（2026-07-12）**：本审计记录的是旧版 Design-only 状态。项目已新增[工程实战主线 v2](工程实战主线-v2.md)与标准库可运行基线，但完整生产实现、真实云部署、外部支付/工单集成和真实学习者效果验证仍未完成，不能宣称生产就绪或保证招聘结果。

生成日期：2026-06-30  
用途：诚实回答“现在的资料和教学步骤到底够不够，哪里还不够”。  
结论：当前项目已经具备从零基础 Design-only 到工业级 Agent 架构判断能力的主线课程前置条件；但它仍然不是已经实现的生产代码框架。

## 1. 总判断

当前可以开始 **Phase 0-11 Design-only 主线教学**。这意味着可以开始教设计、评审、失败推演和验收，不意味着可以直接进入生产运行。

可以肯定的部分：

- 已有 PRD、路线、需求追踪、工业级蓝图、RAG 专项、评测集、Design-only 验收。
- 已明确当前不写代码、不记代码，先训练设计、评审、复述和验收。
- Phase 1-11 已有候选逐课教学手册。
- RAG 专项已经比较像课程，有逐课、实验、评测、契约、安全和长线集成。
- 关键官方资料在 2026-07-01 已做可达性核验并写入证据库。

必须承认的缺口：

- 候选手册还需要通过真实教学不断打磨案例和提问节奏。
- README 曾经承担太多目录职责，阅读路径过长。
- 工业资料池仍需持续补小团队项目、最新 release、license、安全公告和维护状态。
- 还没有进入 Implementation-later，所以没有源码、测试、CI、沙箱、eval runner、观测后端和发布控制台。
- 课程和证据库必须继续分离，避免把 GitHub 热门项目误当可直接采用方案。

## 4. 资料覆盖审计

| 资料类别 | 当前状态 | 需要补入教学的原因 | 代表来源 |
|---|---|---|---|
| 企业 Agent Runtime/Gateway | 薄弱 | 验证 Gateway 不是普通反代，而是身份、工具、策略、观测、评测的治理入口 | [Amazon Bedrock AgentCore](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/what-is-bedrock-agentcore.html)、[AgentCore Gateway](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/gateway.html) |
| 显式工作流与多 Agent | 中 | 需要先教 graph workflow，再教角色协同 | [Microsoft Agent Framework](https://learn.microsoft.com/en-us/agent-framework/overview/)、[Google ADK](https://adk.dev/) |
| 长线任务与 durable execution | 薄弱 | checkpoint、durable workflow、幂等、人工审批不能混为一谈 | [Temporal Docs](https://docs.temporal.io/)、[DBOS](https://docs.dbos.dev/)、[Restate](https://docs.restate.dev/)、[Dapr Workflow](https://docs.dapr.io/developing-applications/building-blocks/workflow/workflow-overview/) |
| MCP 安全 | 薄弱 | MCP 协议不等于企业权限，必须讲 confused deputy、token passthrough、SSRF、session hijacking | [MCP Security Best Practices](https://modelcontextprotocol.io/docs/tutorials/security/security_best_practices) |
| 评测体系 | 中 | 需要把 contract/tool/RAG/memory/long-run/security/cost 分层 | [Ragas](https://docs.ragas.io/)、[DeepEval](https://docs.confident-ai.com/)、[Promptfoo](https://www.promptfoo.dev/docs/intro/)、[Inspect AI](https://inspect.aisi.org.uk/) |
| 可观测 | 中 | OTel/OpenInference 是抽象层，Langfuse/Phoenix 是后端候选 | [Langfuse](https://langfuse.com/docs)、[Phoenix](https://arize.com/docs/phoenix) |
| 记忆专项 | 薄弱 | 必须区分 state/session、memory extraction、retrieval、graph memory | [Mem0](https://github.com/mem0ai/mem0)、[Letta](https://github.com/letta-ai/letta)、Zep/Graphiti |
| Context engineering | 弱 | 不能只讲 prompt，要讲 context packer、tool output trimming、artifact lazy loading、compression、cache alignment | [Google ADK](https://adk.dev/)、[Pydantic AI](https://pydantic.dev/docs/ai/overview/)、[Headroom](https://github.com/headroomlabs-ai/headroom) |

当前真实判断：

- 可以开始 **Phase 0-11 Design-only 主线课**：按阶段教设计、评审、失败推演和验收。
- 不建议跳读，不建议直接进入代码实现。

## 7. 下一步

下一轮优先做：

1. 从 Phase 0 开始正式教学。
2. 每个阶段让学习者产出设计卡、失败样例、评审问题和复述验收。
3. 在教学过程中把薄弱点回流到案例库。
4. 进入代码实现前重新核验外部依赖、license、安全边界和评测工具。
5. 再开启 Implementation-later 工业级框架实现。
