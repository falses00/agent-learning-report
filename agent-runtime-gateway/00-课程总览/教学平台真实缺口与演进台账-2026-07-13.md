## 2. 问题台账

| 优先级 | 问题 | 当前证据 | 解决方向 | 完成定义 |
|---|---|---|---|---|
| P0 | S3-S10 没有可运行工程实验 | 前端阶段为 `implemented: false`；源码 README 只承诺 S0-S2 | 每阶段补 lab、tests、eval、evidence、ADR | 干净环境可一键运行正常、失败、对抗三类检查 |
| P0 | 静态页证据仍是本地自评，不能算工程认证 | 已有五项结构化证据、格式校验、敏感模式拦截和 `self-reported` 导出声明，但浏览器不能复验仓库命令 | 接入 CI 或本地 companion validator，生成可验证 attestation | 证据产物存在、命令结果可重放，签名 manifest 能追溯到 commit |
| 已缓解 | 未实现阶段曾可本地标记通过 | 当前 `implemented: false` 会强制阻断并降级旧状态 | 保留门禁回归测试 | 任意伪造状态都不能把 S3-S10 显示为工程已认证 |
| P0 | 毕业要求与当前规模差距未量化 | 目标 60+ tests、100+ eval；当前 11/5 | 建 graduation gate 与差距仪表 | 自动统计并明确剩余数量与关键缺口 |
| P1 | F0 没有独立实验包 | F0 指向总入口，缺 CLI/API/SQLite/pytest lab | 新建 F0 最小后端实验 | 基础薄弱学习者可独立完成并留证 |
| P1 | S3/S6/S8 缺少专项实验室 | 只有 S5 Memory Lab 有专项门禁 | 优先补 RAG eval、release gate、安全红队 | 三个关键阶段都有可失败、可修复、可回归实验 |
| P1 | 自测题存在两份数据源 | Markdown 简答题与前端选择题分离 | 建稳定 `question_id` 单一题库 | 前端与教材由同一数据生成，CI 阻止漂移 |
| P1 | 内容校验只检查结构 | 未强制速记、岗位映射、证据产物 | 扩展学习内容 schema | 九段教学闭环缺一项即检查失败 |
| P1 | Eval 只检查状态和退款次数 | 缺 trace/audit/citation/security 断言 | 引入可组合 `assertions[]` | critical case 可独立阻塞并输出归因 |
| P2 | 旧 Design-only 文档仍会干扰主线 | 多份旧文档仍写“不写代码” | 标成历史模式，默认入口只展示工程主线 | 新学习者不会误入旧毕业标准 |
| P2 | 前端资料库只展示精选 24 章 | 大量参考资料未被分类索引 | 明示“精选入口”，另建全量证据索引 | 用户知道哪些必学、哪些参考、哪些历史 |

## 3. 当前外部能力基线

以下一手资料共同说明，工业 Agent 教学必须超越 prompt 和简单 tool calling：

- [OpenAI Agents SDK](https://platform.openai.com/docs/guides/agents-sdk)：agent loop、handoff、guardrails、state、tracing 与 approval。
- [OpenAI Remote MCP](https://platform.openai.com/docs/guides/tools-remote-mcp)：远程 MCP 工具、审批与恶意 server 数据外泄边界。
- [MCP Specification](https://modelcontextprotocol.io/specification/2025-06-18)：host/client/server、capability negotiation、resources、prompts、tools。
- [MCP Security Best Practices](https://modelcontextprotocol.io/docs/tutorials/security/security_best_practices)：token passthrough、confused deputy、SSRF、session hijacking。
- [LangGraph Persistence](https://docs.langchain.com/oss/python/langgraph/persistence)：checkpoint、human-in-the-loop、time travel、fault tolerance 与跨线程 store。
- [Temporal Human-in-the-Loop AI](https://docs.temporal.io/ai-cookbook/human-in-the-loop-python)：durable workflow 中等待并恢复人工审批。
- [OpenTelemetry GenAI Semantic Conventions](https://github.com/open-telemetry/semantic-conventions-genai)：model、agent、MCP 的跨平台 trace/metric/event 语义。
- [OWASP Top 10 for LLM Applications 2025](https://genai.owasp.org/llm-top-10/)：prompt injection、敏感信息泄漏、供应链、excessive agency、向量与嵌入风险。
- [NIST AI RMF Generative AI Profile](https://www.nist.gov/publications/artificial-intelligence-risk-management-framework-generative-artificial-intelligence)：风险治理、映射、测量与管理。

外部资料只用于校准能力边界。任何框架、论文或云产品都不能替代本项目自己的失败测试和验收证据。

### Cycle 3：补关键工业门禁

- [ ] 建 S4 crash/resume 与 query-before-retry。
- [ ] 建 S6 critical release gate。
- [ ] 建 S8 prompt injection、SSRF、egress 与恶意 MCP 红队套件。

### Cycle 4：毕业与作品集

- [ ] 建毕业差距统计脚本。
- [ ] 建 release manifest、canary、rollback drill。
- [ ] 建可导出的作品集证据包与现场答辩清单。

## 5. 发布原则

每个 Cycle 只有同时满足以下条件才能发布：

1. 项目内校验脚本通过。
2. 目标功能在真实浏览器或真实 CLI 中运行。
3. 至少一个失败或对抗探针通过。
4. 未实现边界在 UI 与文档中明确可见。
5. 变更已经过独立 review，且没有把教学模拟冒充生产能力。
