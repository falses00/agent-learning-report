新手先读这 5 个入口，不要一开始读 GitHub JSON 快照：

3. [Phase-00.1-半懂起步教学手册](../02-阶段教学手册/Phase-00.1-半懂起步教学手册.md)：正式第一课。
4. [每节课互动模板](每节课互动模板.md)：知道以后怎么问、怎么复述、怎么卡住时求助。
5. [Design-only产物包模板](Design-only产物包模板.md)：知道每节课要交什么。

## 2. 学习总规则

当前默认是 Design-only：

- 不写代码。
- 不记代码。
- 每阶段产出设计卡、失败场景、评审问题、复述验收。
- Implementation-later 是后续工程化，不作为当前通过条件。

每阶段固定学习循环：

```text
理解失控风险
-> 看系统边界
-> 设计最小契约或门禁
-> 推演失败场景
-> 做架构评审
-> 写复述验收
-> 记录复述评分
-> 判断是否进入下一阶段
```

每阶段产物统一使用：

- [Design-only产物包模板](Design-only产物包模板.md)
- [学习者复述评分记录](学习者复述评分记录.md)

## 4. 学习可视化前端怎么嵌入主线

[学习可视化前端 PRD](../01-产品需求与路线/学习可视化前端PRD.md) 不是新的主线课程，也不是生产控制台。它只负责把抽象概念变成可看、可点、可复述的学习辅助。

当前嵌入规则：

| 主线阶段 | 前端辅助什么 | 不能替代什么 | 过关信号 |
|---|---|---|---|
| Phase 0.1 | 用退款工单看懂“建议和执行分离” | 不能替代第一节课复述 | 能说出为什么 ToolCall 是申请 |
| Phase 2 | 用时间线看懂 Run/Step/Checkpoint | 不能替代运行时状态机设计卡 | 能说明聊天记录为什么不是状态 |
| Phase 3 | 用链路图看懂 Gateway/Policy/Audit | 不能替代网关边界图 | 能说明 prompt 为什么不能决定权限 |
| Phase 4 | 用 checkpoint/resume 时间线看懂长线任务可靠性 | 不能替代长线任务设计卡和幂等推演 | 能说明 retry 为什么不能重复副作用 |
| Phase 6 | 用记忆生命周期看懂写入、召回、撤销、过期和审计 | 不能替代记忆写入门禁设计 | 能说明错误事实如何撤销并防止再次召回 |
| Phase 7 | 用 Eval 门禁页看懂评测阻塞 | 不能替代评测集设计 | 能判断哪些失败必须阻塞发布 |
| Phase 8 | 用 Trace 回放页看懂 span/audit/event | 不能替代观测字段表 | 能从失败定位到具体层 |
| Phase 9 | 用工具风险矩阵和 sandbox profile 看懂安全隔离 | 不能替代威胁模型和沙箱策略设计 | 能按工具风险选择隔离或人工审批 |
| Phase 10 | 用多 Agent handoff 图看懂协同边界 | 不能替代 handoff schema 和分歧合并规则 | 能说明多 Agent 分歧为什么按证据合并 |
| Phase 11 | 用 Release 学习模拟看懂版本组合 | 不能替代 release audit packet | 能说明发布不是只发代码 |
| RAG 专项 | 用 RAG 问题诊断器看懂问题、指标、优化、eval case | 不能替代 RAG 评测集和安全多租户设计 | 能说明检索命中为什么不等于可信答案 |
| LLM-Wiki 知识层 | 用来源审核、知识卡和版本记录看懂资料如何进入课程 | 不能替代人工审核、license 复核和课程重写 | 能说明 star 高、LLM 生成或检索命中都不等于可信知识 |

使用前端辅助时，每次仍要完成：

- Design-only 产物包。
- 失败路径推演。
- 复述评分记录。
- 阶段验收清单。
- 中文优先双语检查：主要学习文案必须中文，关键工程术语保留英文括注。
- 低分回流检查：复述低于 3 分时，必须生成重讲任务或内容修订目标。
- Codex 协作检查：每次学习任务必须能看到增强提示词、推荐 subagent、期望证据和主线程整合规则。
- 知识层检查：外部资料必须能追溯到 KnowledgeSource、SourceReview、KnowledgeCard、KnowledgeVersion 和 ImportQueue。

如果可视化让学习者误以为已经有真实生产系统、真实发布、真实工具执行或真实企业数据，则该可视化直接判失败。

RAG 不作为主线替代品，而是插入 Phase 3、5、6、7、8、9、11。

4. [RAG评测数据集设计](../22-评测集/RAG评测数据集设计.md)

RAG 学习原则：

- 先复现失败，再选优化。
- 先设计指标，再谈上线。
- 先过权限、引用、评测、安全，再谈效果。

## 6. 证据库怎么读

`10-GitHub项目调研` 是证据库，不是新手教材。

读法：

- 课程提到某个外部项目时，再去查证据索引。
- 不要按 JSON 快照逐个读。
- star 只表示关注度，不表示企业可用。
- 任何生产采用都要重新核验 license、安全、维护状态和版本。

但不能假装：

- 这不是已经实现的生产代码框架。
- GitHub 项目和外部依赖在进入代码阶段前仍要重新核验 release、license、安全公告和维护状态。
- LLM-Wiki 相关资料只能作为知识治理模式和分类参考，不代表可以复制外部内容。
- 真实工程还缺源码、测试、CI、沙箱、评测 runner、观测后端和发布控制台。
- 06/07/10 已有大量资料，但必须先通过索引使用。

所以当前最正确的学习动作是：

1. 从 Phase 0 开始讲课程地图、能力地图、缺口审计。
2. 按 Phase 1-11 顺序学习每份阶段手册。
3. 每阶段只做 Design-only 产物和复述验收。
4. 不跳到代码实现，不把文档误认为生产就绪。
5. 进入 Implementation-later 前，再做一次资料、依赖、安全和评测复核。
