# 唯一学习入口

这份文件是课程的唯一入口。其他路线图、前端设计稿、GitHub 快照和历史核验报告都不能改变这里的顺序。

## 先判断起点

在 PowerShell 中运行：

```powershell
python --version
git --version
docker --version
```

然后独立完成下面四件事：

- 写一个 Python 函数并用 `pytest` 测试。
- 调用一个 HTTP JSON API，能解释状态码、超时和重试。
- 用 SQL 建表、写入、查询并说明事务的作用。
- 用 Git 创建分支、提交并解释如何回退一个错误变更。

若其中两项以上做不到，先进入“基础补齐”；否则直接进入工程主线。不要用 Agent 框架掩盖编程基础缺口。

## 两条前置路径

| 路径 | 适合谁 | 目标 | 通过证据 |
|---|---|---|---|
| F0 基础补齐 | Python/HTTP/SQL/Git 不稳 | 具备后端实验最低能力 | CLI 小程序、API、SQLite、pytest、Git 提交 |
| F1 Agent 直觉 | 会编程但不了解 Agent | 区分模型、Runtime、Tool、Policy、Memory、Eval | 能解释退款为何必须“建议与执行分离” |

F1 可使用[第一节课](第一节课-用工单故事理解工业级Agent.md)和[术语表](术语表.md)，但最多用 1-2 天。之后立即进入可运行实验。

## 每周学习循环

1. **定义**：一句话说明本周知识点，并画清责任边界。
2. **复现**：先让一个代表性故障真实发生。
3. **实现**：写最小代码修复，不引入无关框架。
4. **验证**：跑正常、失败、对抗三类检查。
5. **观察**：保存 trace、audit、eval 或性能证据。
6. **解释**：回答五道自测题，讲清取舍和剩余风险。
7. **沉淀**：把故障加入 regression set，把决策写入 ADR。

任一周没有“可运行增量 + 失败路径 + 验收证据”，不得标记完成。

## 教学提示词

```text
我要学习 OpsPilot 的【阶段/知识点】。我的当前水平是【基础/半懂/可独立编码】。

请一次性完成：
1. 用一句话定义知识点，并列出 3-6 个关键概念；每个概念按“白话解释 -> 为什么存在 -> 缺少会发生什么事故 -> 如何用工程证据验收”讲解。
2. 指明它在 OpsPilot 当前代码中的输入、输出、责任边界和不负责什么。
3. 先给出一个可复现的真实失败案例，再带我完成最小实现；不要只给最终代码。
4. 给出正常、失败、对抗三类测试，以及应该看到的 trace/audit/eval 证据。
5. 列出常见错误、排查顺序、修复方法和修复后必须加入的回归用例。
6. 给我 5 道偏设计判断和故障分析的自测题；我答错时指出具体误区并用更简单的例子重讲。
7. 最后生成一页速记清单、下一阶段预习清单，并按课程门禁判断我能否继续。

限制：本轮只处理当前阶段；所有结论优先引用项目代码、测试和官方一手资料；明确区分课程基线、生产可用和仍未实现的部分。
```

## 停止条件

出现下列任一情况时，先补缺口，不继续堆功能：

- 只能复制代码，不能解释边界和失败模式。
- 测试只覆盖 happy path。
- 工具写操作没有 schema、policy、approval 或 idempotency。
- RAG 没有引用、租户过滤和离线评测。
- 修改 prompt/model/tool 后没有跑回归集。
- trace 中出现 secret、PII 或无法关联 `run_id`。
- 多 Agent 没有证明优于单 Agent 基线。
