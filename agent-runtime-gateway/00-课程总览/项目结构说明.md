# 项目结构说明

生成日期：2026-06-29  
用途：解释本项目为什么这样分文件夹，方便按课程顺序学习。

## 1. 根目录

| 路径 | 用途 |
|---|---|
| `README.md` | 默认入口，兼容 GitHub、编辑器和文档工具 |
| `00-课程总览` | 先读这里，理解课程地图、能力地图、缺口审计、教学循环、术语和项目结构 |
| `01-产品需求与路线` | 定义要做什么、为什么做、路线如何变化、需求如何验收 |
| `02-阶段教学手册` | 每个阶段怎么跟做、怎么验收 |
| `03-设计体系` | 学企业级 Agent 的分层设计方法 |
| `04-测评审核体系` | 学如何证明准确率、安全性、记忆质量、多 Agent 协作质量 |
| `05-实用化落地` | 学如何把课程 demo 推进到 MVP-practical 和内部试点 |
| `06-工业级框架蓝图` | 学工业级 Agent Framework 的不可妥协标准、长线运行和审计门禁 |
| `07-RAG问题诊断与优化` | 学如何按问题诊断 RAG，并选择 chunking、rewrite、rerank、GraphRAG、安全、评测等方案 |
| `08-学习可视化前端设计` | Product Design / Open Design 前端设计包，包含线框、视觉方向图、中文化和学习迭代规范 |
| `10-GitHub项目调研` | 存放外部项目调研、采用准则和 GitHub 快照 |
| `22-评测集` | 存放 RAG、记忆、轨迹、安全等评测集设计和示例数据 |

## 2. 为什么用数字前缀

数字不是为了显得复杂，而是为了让文件夹在资源管理器里自然排序：

```text
00 先看总览
01 再看产品和路线
02 跟着阶段手册实施
03 学设计体系
04 学测评审核体系
05 学实用化落地
  06 学工业级框架蓝图
  07 学 RAG 问题诊断与优化
  08 学习可视化前端设计
  10 查外部项目调研
```

当前已创建 `22-评测集`，因为评测样例本身属于设计与验收资产，不要求先进入代码阶段。

后续如果开始写代码，可以继续增加：

```text
20-源码
21-测试
23-运行脚本
30-实验报告
90-归档
```

当前还没有进入代码阶段，所以暂时不创建空代码目录，避免目录太多反而难懂。

## 4. 目录维护规则

- 教学总路线放在 `00-课程总览`。
- 课程地图、能力地图、缺口审计和文件结构重构建议放在 `00-课程总览`。
- 产品边界、路线变化、需求矩阵和 ADR 放在 `01-产品需求与路线`。
- 学习可视化前端的产品需求和设计边界放在 `01-产品需求与路线`，实现工程后续再单独建目录。
- 学习可视化前端的 Product Design / Open Design 设计包放在 `08-学习可视化前端设计`，它不是源码目录，不代表已经进入前端实现。
- v0.4-pre 原型前冻结包、页面状态矩阵、mock 数据字典、学习验证脚本、v0.4 brief 回放、首轮原型任务说明和前置完成审计记录也放在 `08-学习可视化前端设计`，用于进入 Product Design 原型前验收。
- v0.4 静态低保真可点击原型放在 `08-学习可视化前端设计/v0.4-低保真可点击原型`，它是教学原型，不是前端工程目录。
- LLM-Wiki 知识层的前端表达和可维护扩展方案放在 `08-学习可视化前端设计`；外部项目证据和采用风险放在 `10-GitHub项目调研`。
- 某个阶段的跟做步骤放在 `02-阶段教学手册`。
- 抽象设计方法放在 `03-设计体系`。
- 所有评测、审核、红队、发布门禁放在 `04-测评审核体系`。
- 从教学到内部试点的落地计划放在 `05-实用化落地`。
- 工业级框架不可妥协标准、外部框架映射、能力基线、运行时状态机、长线运行、多 Agent 协同、分层契约和发布门禁放在 `06-工业级框架蓝图`，入口是 `00-蓝图索引.md`。
- RAG 的问题诊断、优化矩阵、小项目实验、评测回归、安全多租户和长线 Agent 集成放在 `07-RAG问题诊断与优化`，入口是 `00-RAG索引.md`。
- 外部资料、GitHub 项目、快照和采用准则放在 `10-GitHub项目调研`，入口是 `00-证据索引.md`。
- 外部资料进入课程前必须先经过 SourceReview、KnowledgeCard、KnowledgeVersion 和 ImportQueue 规则，不直接进入课程正文。
- 评测集设计、JSONL 示例、未来 golden/red team/trajectory 数据集放在 `22-评测集`。
- 不把大文件、缓存、依赖、实验输出放进本项目根目录。
