# 半懂学习者导航

生成日期：2026-07-01  
用途：给“不想写代码、不想记代码、但想通过和 Codex 对话学会工业级 Agent 设计”的学习者一个低压入口。

## 1. 先把目标说人话

你现在不是要背会 LangGraph、MCP、RAG、Temporal、Promptfoo 这些名字。

你真正要学会的是：

```text
当一个 Agent 要替人做长期任务时，
我知道它会在哪些地方失控；
我知道系统应该分哪些层防住这些失控；
我知道怎么让 Codex 帮我设计、评审、验证和复盘；
我能判断一个方案是 demo、内部试点，还是接近工业级。
```

所以你不是来当“背 API 的人”，而是来训练“会问、会审、会验收的 Agent 架构负责人”。

## 2. 你现在最容易卡住的地方

| 卡点 | 为什么会卡 | 本课程怎么处理 |
|---|---|---|
| 名词太多 | Runtime、Gateway、Policy、Trace、Eval 很像一团词 | 先用工单故事串起来 |
| 不知道先读哪个文件 | README、课程地图、PRD、蓝图都像入口 | 先读本文件，再读第一节课脚本 |
| 不会判断自己懂没懂 | 看完文档感觉懂，复述时说不出来 | 每节课都用填空复述卡 |
| 容易被 GitHub 项目带跑 | star 多看起来就像正确答案 | 先学采用准则，再看项目 |
| 不写代码怕学不实 | 以为不写代码就只是听概念 | 每阶段必须产出设计卡、失败卡、评审问题 |
| 不知道怎么问 Codex | 只会说“继续”，得到的内容可能太抽象 | 使用固定提问模板 |
| 不知道什么叫工业级 | 容易把能跑的 demo 当成可用系统 | 每阶段都对照失控风险和验收门禁 |
| 长线任务很抽象 | checkpoint、resume、idempotency 不直观 | 用“任务做到一半断了”来讲 |

## 4. 最小学习地图

你可以把整套课先记成 5 个问题：

| 问题 | 对应阶段 | 你要学会的判断 |
|---|---|---|
| Agent 要做什么，边界在哪里？ | Phase 0-1 | 需求和契约是否清楚 |
| Agent 怎么长期运行不断片？ | Phase 2-4 | 状态、恢复、重试是否可靠 |
| Agent 能调用什么，能记什么？ | Phase 5-6 | 工具和记忆是否受治理 |
| Agent 怎么证明自己可靠？ | Phase 7-8 | 评测、trace、audit 是否闭环 |
| Agent 怎么安全协同和上线？ | Phase 9-11 | 沙箱、多 Agent、发布治理是否成立 |

如果你能先把这 5 个问题讲清楚，后面的名词就不会散。

## 5. 每节课你只需要交 4 个东西

你不写代码，但每节课要交这 4 个非代码产物：

| 产物 | 你要写什么 | 例子 |
|---|---|---|
| 一句话解释 | 本阶段到底防什么失控 | “Phase 3 防止 Agent 绕过工具权限” |
| 坏设计识别 | 如果偷懒会怎样 | “只靠 prompt 禁止删除账号是不够的” |
| 设计卡 | 系统应该加哪一层 | “Tool Gateway 必须校验 schema 和 policy” |
| 复述验收 | 你用自己的话讲一遍 | “工具调用是申请，不是执行” |

## 6. 每次你可以这样问我

```text
继续 Phase X。
我半懂不懂，不想写代码，也不想记代码。
请先调用合适的 subagent 增强提示词。
然后用一个具体业务故事讲这一阶段；
先讲人话，再给术语；
每个术语都要说明为什么存在、少了会出什么事故；
最后给我一个填空式复述卡和 3 个追问。
如果我复述不清楚，请不要进入下一阶段。
```

## 7. 我应该怎样教你

每节课我都应该按这个顺序教：

```text
业务故事
-> 失控画面
-> 为什么普通聊天 demo 不够
-> 需要哪一层
-> 这一层的白话解释
-> 这一层的专业术语
-> 坏设计案例
-> 设计卡
-> 复述卡
-> 是否进入下一阶段
```

如果我一上来就讲框架名、工具名、论文名，你可以打断我：

```text
先别讲名词，先用工单故事解释。
```

## 8. 你怎样判断自己真的懂了

不是看你能不能复述定义，而是看你能不能回答：

| 检查问题 | 如果你答不出 |
|---|---|
| 这一层防什么事故？ | 说明你只记住了名词 |
| 如果没有这一层会怎样？ | 说明你不知道它为什么存在 |
| 它和上一层有什么边界？ | 说明你还没形成系统图 |
| 哪个失败必须阻塞下一步？ | 说明你还不会验收 |
| 我能让 Codex 怎么实现或检查它？ | 说明你还不能指挥实现 |

## 9. 卡住时怎么办

卡住时不要硬读文档，直接用这三个问题：

```text
1. 用一个现实类比重新讲。
2. 给我一个坏设计事故。
3. 给我一个填空复述模板。
```

如果还是不懂，就把阶段缩小：

```text
本轮只讲一个概念，不进入下一节。
```
