# 能力地图

生成日期：2026-06-30  
用途：把“工业级 Agent 需要哪些能力”映射到阶段、文档、练习、失败样例和验收证据。

## 1. 使用方式

当你不知道某个能力该在哪学时，看这张表：

- 能力域：你要掌握的架构能力。
- 对应阶段：主线课程位置。
- 练习：当前 Design-only 要做什么。
- 失败样例：用来判断你是否真正理解。
- 后续工程证据：Implementation-later 才需要。

## 2. 工业能力映射

| 能力域 | 对应阶段 | 当前核心文档 | Design-only 练习 | 失败样例 | 后续工程证据 |
|---|---|---|---|---|---|

## 3. 当前薄弱能力

| 薄弱能力 | 为什么薄弱 | 补齐动作 |
|---|---|---|
| 案例库深度 | Phase 1-11 已补候选手册，但真实教学案例还要积累 | 每阶段扩展 3 类案例：正常、困难、失败 |
| Durable execution | 现在偏 LangGraph checkpoint，缺 Temporal/DBOS/Restate/Dapr 对照 | 新增长线任务对照课 |
| Context engineering | 当前强调 prompt，不够强调上下文组装链路 | 新增 context packer 教学 |
| MCP 安全 | Phase 5 已加入攻击路径，但还需要更多真实 server 准入案例 | 扩充 MCP server case 库 |
| Memory 项目对比 | Phase 6 已纳入 Mem0/Zep/Letta/Graphiti 方向，但缺逐项目对照表 | 新增 memory 项目对比 |
| Eval taxonomy | Phase 7 已补分层矩阵，但缺更多可复用 case | 扩展 golden/redteam/regression 样例库 |
| 工程实现证据 | 当前仍是 Design-only，没有源码和 CI 证据 | 进入 Implementation-later 后补源码、测试、eval runner、trace/audit、sandbox |

## 4. 过关复述模板

每学完一个能力域，你应该能说：

```text
这个能力解决 [失控风险]。
它位于 [系统层级]。
我设计了 [契约/门禁/评测/trace]。
如果它缺失，会出现 [失败样例]。
当前 Design-only 证据是 [设计资产]。
后续 Implementation-later 证据是 [工程验证]。
```

说不出这段话，就不要急着进入下一阶段。
