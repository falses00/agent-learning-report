# 开始这里 - 半懂版

生成日期：2026-07-01  
用途：当你打开项目后不知道先读什么时，只按这一页走。

## 1. 今天只做三件事

不要先读完整路线，不要先读 GitHub 调研，也不要先读 RAG 专项。

今天只做：

1. 读 [半懂学习者导航](半懂学习者导航.md)：知道自己要学的不是代码，而是设计、评审和验收。
2. 上 [第一节课-用工单故事理解工业级Agent](第一节课-用工单故事理解工业级Agent.md)：用退款工单理解工业级 Agent 为什么不是聊天机器人。
3. 用 [每节课互动模板](每节课互动模板.md)：让 Codex 按你的理解程度讲，不懂就停下来修复。

## 2. 今天不要读这些

这些文件先只知道存在：

| 暂时不要读 | 原因 |
|---|---|
| GitHub JSON 快照 | 证据库，不是教材 |
| RAG 全套专项 | 会抢走 Phase 1 主线注意力 |
| 全部 Phase 2-11 手册 | 还没建立第一层直觉 |
| 工业级蓝图全部文件 | 适合作为后续查证，不适合第一眼读 |
| 外部框架调研 | 先学判断，再看项目 |

## 3. 今天的过关标准

今天不要求你懂所有名词，只要求你能讲清：

```text
为什么“建议退款”和“执行退款”必须分开？
为什么只保存聊天记录不等于长线任务状态？
为什么一个能回答问题的 Agent 还不能直接上线？
```

如果这三个问题还讲不清，不进入 Phase 1。

## 4. 你可以直接这样开始

把这段发给 Codex：

```text
开始 Phase 0.1：半懂起步。

我半懂不懂，不想写代码，也不想记代码。
请先调用合适的 subagent 增强提示词。
然后只用工单退款故事讲清工业级 Agent 和聊天 demo 的区别。
不要扩展框架名，不要进入代码。
最后用填空题考我，判断我能不能进入 Phase 1。
```
