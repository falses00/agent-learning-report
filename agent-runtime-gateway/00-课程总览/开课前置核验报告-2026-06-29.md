## 4. 多轮前置反思

### 第一轮：目标与范围

结论：课程目标是构建企业级 Agent Runtime/Gateway，不是聊天机器人 demo。  
风险：如果直接写代码，会把架构、评估、安全和记忆治理全混在一起。  
处理：先完成 PRD、路线、设计体系、测评审核体系和调研证据。

### 第二轮：资料最新性

结论：2026-06-28 的 GitHub 快照已不够，必须刷新。  
处理：新增 `GitHub核心项目快照-2026-06-29.json` 和 `GitHub补充项目快照-2026-06-29.json`。  
边界：GitHub stars、release、license、README 都会变化；当前结论仅代表 2026-06-29 核验时点。

### 第三轮：工程可验证性

结论：课程不能只讲概念，必须每阶段都有正常路径和失败路径。  
处理：新增 `测评审核体系.md`，覆盖答案、工具、轨迹、记忆、Gateway、安全、多 Agent、可观测性。

### 第四轮：教学可复现性

结论：文件夹不能只按文档类型分，要按学习顺序分。  
处理：根目录改为 `00-课程总览`、`01-产品需求与路线`、`02-阶段教学手册`、`03-设计体系`、`04-测评审核体系`、`10-GitHub项目调研`。

### 第五轮：现实价值

结论：这门课真正有价值的地方不是“会写 Agent”，而是会把 Agent 放进可靠企业系统。  
高价值能力：Runtime、Gateway、Tool Governance、Memory Eval、Red Team、Observability、Sandbox、Governance。

### 第六轮：工业级可用性

结论：只靠 PRD 和路线还不够，必须把“可用”拆成能力基线、状态机、发布门禁、SLO、威胁模型、MCP 准入、多 Agent 协同契约和记忆生命周期。  
处理：新增 `06-工业级框架蓝图` 下的能力基线、状态机、发布门禁、外部框架映射、多 Agent 协同契约，并把 SRE、发布、测评、数据模型文档接入这些门禁。  
边界：这些是工业级框架前置设计，不等于已经有生产代码；进入代码阶段后仍必须逐项实现和验证。

### 第七轮：最新资料与外部项目

结论：GitHub 热门项目可以提高路线先进性，但不能用 star 数替代工程判断。  
处理：2026-06-30 由 researcher subagent 先增强提示词并核验 LangGraph、Microsoft Agent Framework、OpenAI Agents SDK、Temporal、Mem0、Letta、Langfuse、Promptfoo、E2B、OpenHands、MCP、Composio 等项目，再由主线程生成 GitHub CLI 快照。  
边界：本轮只做仓库元数据和 README/定位级判断，没有做源码级安全审计。
