# 需求追踪矩阵

生成日期：2026-06-28  
项目：Agent Runtime Gateway  
用途：把 PRD 需求映射到参考项目、实施阶段和验收标准。

## 1. 优先级定义

| 优先级 | 含义 |
|---|---|
| P0 | MVP-complete 必须实现，否则产品定位不成立 |
| P1 | Beta 必须实现，增强企业可用性 |
| P2 | GA/长期增强，视场景推进 |

补充说明：

- `MVP-slice`：Phase 3 结束时形成最小端到端演示，证明 Runtime + Gateway + Tool + Policy + Audit 链路可行。
- `MVP-complete`：Phase 9 结束时 P0 能力基本打通，包含 runtime、gateway、checkpoint、MCP、memory、eval、observability、sandbox。

参考模式：

- `dependency allowed`：可作为候选依赖继续评估。
- `adapter candidate after license review`：可做 adapter 候选，但必须先做许可证/商业条款审查。
- `pattern only`：只借鉴设计模式，不引入依赖。

## 3. MVP 验收

### 3.1 MVP-slice 验收

Phase 3 结束时只需要证明下面这条最小链路：

```text
User Request
-> Request Gateway
-> Minimal Runtime
-> Tool Gateway
-> Policy Decision
-> Mock Tool
-> Audit Event
```

必须通过：

- 一个正常读工具调用。
- 一个越权写工具调用被拒绝。
- 一个格式错误 ToolCall 被 schema 拦截。
- 一个 prompt injection 工具参数被拒绝。
- 一个 audit event 能复盘请求、主体、工具、决策和结果。

### 3.2 MVP-complete 验收

Phase 9 结束时证明下面这条完整链路：

```text
User Request
-> Request Gateway
-> Agent Runtime
-> Tool Gateway
-> Policy Decision
-> MCP-like Tool Call
-> Checkpoint
-> Trace/Audit
-> Eval Gate
-> Sandbox
```

必须通过：

- 一个正常读工具调用。
- 一个越权写工具调用被拒绝。
- 一个格式错误 ToolCall 被 schema 拦截。
- 一个任务中断后 resume。
- 一个重复 operation_id 不产生重复副作用。
- 一个 prompt injection redteam 用例被拦截。
- 一个 trace 能复盘完整链路。
