# 教学到工程交付映射

生成日期：2026-06-29  
用途：把每个教学阶段对应到真实工程交付，确保课程不是空讲。

## 1. 映射原则

每一课都必须同时产生：

- 教学理解：你知道为什么做。
- 设计资产：设计决策卡、失败场景、架构评审问题、验收清单。
- 工程资产：文档、schema、代码、测试、配置或报告。当前 Design-only 模式下可先不写代码。
- 验收证据：命令、测试、trace、eval、review，或设计阶段的评测样例、失败推演、门禁表。
- 实用价值：能被后续阶段复用，或能阻止真实风险。

## 2. 教学到架构掌握映射

| 阶段 | 你不写代码时要掌握 | 设计资产 | 评审问题 |
|---|---|---|---|
| Phase 1 契约层 | schema 为什么是边界 | 字段责任卡、坏输入表 | 哪些模型输出不能进入工具层 |
| Phase 2 最小运行时 | Run/Step/State 为什么不是聊天记录 | 状态机图、失败恢复卡 | 中断后从哪里恢复 |
| Phase 3 网关治理 | prompt 不能决定权限 | Gateway 边界图、工具风险卡 | Agent 能否绕过 Tool Gateway |
| Phase 4 长线任务 | checkpoint、resume、幂等 | 恢复流程图、重复副作用卡 | 重试会不会重复执行写操作 |
| Phase 5 MCP 工具 | 协议不等于治理 | 工具准入卡、凭据隔离卡 | MCP server 失控时如何禁用 |
| Phase 6 记忆 | 记忆写入必须有门禁 | 记忆生命周期表 | 错误事实如何撤销 |
| Phase 7 测评审核 | 评估不只看最终答案 | golden/red team case 设计 | 哪些失败必须阻塞发布 |
| Phase 8 可观测 | trace 是复盘证据 | span 设计、audit 字段 | 出错时能否定位到层级 |
| Phase 9 安全隔离 | 高风险执行必须隔离 | sandbox profile、egress 策略 | 秘钥和文件如何不被拿走 |
| Phase 10 多 Agent | 协作必须有 handoff 和锁 | handoff schema、资源锁表 | 两个 agent 冲突谁仲裁 |
| Phase 11 治理发布 | 上线必须可回滚 | release gate、rollback card | 版本退化如何撤回 |

## 3. 阶段工程映射表

| 阶段 | 教学内容 | 工程交付 | 真实应用价值 |
|---|---|---|---|
| Phase 1 契约层 | schema 是第一道边界 | `schemas/`、validation tests、bad cases | 防止模型输出直接进入工具层 |
| Phase 2 最小运行时 | Agent loop、Run/Step/State | `runtime/`、mock model、mock tool、trace JSON | 让 Agent 行为可复现 |
| Phase 3 网关治理 | 请求、模型、工具、策略、审计 | `gateway/`、policy、audit log | 控制谁能做什么 |
| Phase 4 长线任务 | checkpoint、resume、HITL | persistence、resume tests、approval state | 任务不中断、不重复副作用 |
| Phase 5 MCP 工具 | tool registry、adapter、凭据 | MCP adapter、tool lifecycle tests | 外部工具可治理接入 |
| Phase 6 记忆 | 写入、召回、遗忘、隔离 | memory store、retrieval policy、memory eval | 长期上下文可信 |
| Phase 7 测评审核 | golden/red team/trajectory | eval cases、CI gate、reports | 证明质量不退化 |
| Phase 8 可观测 | trace/span/replay/cost | telemetry schema、replay command | 失败可定位 |
| Phase 9 安全隔离 | sandbox、网络/文件限制 | sandbox profile、adversarial tests | 高风险工具不裸奔 |
| Phase 10 多 Agent | handoff、共享状态、仲裁 | role contracts、handoff tests | 协作不失控 |
| Phase 11 治理发布 | release gate、rollback、audit | release checklist、approval flow | 可上线、可回滚 |
