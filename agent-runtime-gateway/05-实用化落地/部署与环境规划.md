# 部署与环境规划

生成日期：2026-06-29  
目标：定义从本地教学到内部试点的环境路线。

## 1. 环境分层

| 环境 | 用途 | 技术形态 |
|---|---|---|
| `local-dev` | 本地教学、开发、测试 | Python venv + SQLite 或本地文件 |
| `demo` | 给用户演示完整链路 | Docker Compose + SQLite/PostgreSQL |
| `prod-lite` | 内部低风险试点 | Docker Compose + PostgreSQL + OTel 可选 |

MVP 不上 Kubernetes，不做多云，不做复杂服务网格。

## 2. local-dev

目标：

- 30 分钟内跑通。
- 不需要真实外部账号。
- 不需要真实生产凭据。
- mock model 和 mock tool 可离线运行。

必须提供：

- 安装命令。
- 启动命令。
- 示例工单。
- 一键评测命令。
- trace/replay 命令。

## 3. demo

目标：

- 可以给团队演示端到端链路。
- 可以切换 mock model 和真实模型 provider。
- 数据仍然低风险。

组件：

- app service。
- SQLite 或 PostgreSQL。
- 可选 OTel collector。
- 可选本地 dashboard，不作为 MVP 必需。

## 4. prod-lite

目标：

- 内部小范围试点。
- 只读工具。
- 人工确认关键动作。
- 有审计和回滚。

必须补齐：

- PostgreSQL。
- 配置冻结。
- 环境变量规范。
- 备份策略。
- 日志保留。
- 评测门禁。
- rollback checklist。

## 5. 配置原则

所有环境必须区分：

- `APP_ENV`
- `DATABASE_URL`
- `MODEL_PROVIDER`
- `MODEL_API_KEY`
- `MAX_STEPS_PER_RUN`
- `RUN_TIMEOUT_SECONDS`
- `TOKEN_BUDGET_PER_RUN`
- `ALLOW_WRITE_TOOLS`
- `TRACE_EXPORT_MODE`

密钥不得写入 prompt、日志、评测集和 Git 仓库。

## 6. 部署验收

demo 环境验收：

- 服务能启动。
- 示例工单能跑通。
- trace 可查看。
- audit 可查看。
- eval 可运行。
- 越权写操作被拒绝。

prod-lite 环境验收：

- 通过 demo 验收。
- 数据库迁移可重复执行。
- 配置有版本记录。
- 回滚流程可演练。
- 低风险用户试点范围明确。
