# 发布门禁与回滚计划

生成日期：2026-06-29  
目标：让每次从教学 demo 到内部试点的变更都有门禁和回滚。

## 0. 与工业级发布门禁的关系

本文件是落地操作计划；完整标准见：

- `H:\Creat A Agent\agent-runtime-gateway\06-工业级框架蓝图\工业级发布门禁矩阵.md`
- `H:\Creat A Agent\agent-runtime-gateway\06-工业级框架蓝图\测评审核门禁矩阵.md`
- `H:\Creat A Agent\agent-runtime-gateway\06-工业级框架蓝图\生产就绪审核清单.md`

从 beta 开始，发布对象必须作为一个版本组合包管理，不能只记录代码 commit。

## 1. 发布对象

每次发布必须记录：

- Agent version。
- Prompt version。
- Model/provider version。
- Tool registry version。
- Policy version。
- Memory schema/retrieval config version。
- Eval set version。
- Sandbox profile version。
- Runtime/Gateway version。
- Migration version。

## 2. 发布前门禁

必须通过：

- schema tests。
- unit tests。
- integration smoke tests。
- red team critical cases。
- memory leakage tests。
- tool policy tests。
- checkpoint/resume/retry/cancel tests。
- operation_id idempotency tests。
- sandbox egress/resource tests。
- trace/audit coverage check。
- eval pass threshold。
- release evidence packet review。

不同变更类型的门禁以 `工业级发布门禁矩阵.md` 为准。新增工具、policy、memory、runtime、sandbox 和迁移变更默认按高风险处理。

## 4. 发布等级

| 等级 | 允许范围 | 要求 |
|---|---|---|
| docs-only | 文档和教学路线 | 链接、路径、术语一致 |
| experimental | 本地实验 | 契约测试和明确不可生产标记 |
| beta | 内部试点 | eval、trace/audit、回滚路径完整 |
| production-candidate | 小范围生产候选 | SLO、告警、事故响应和恢复演练 |
| production | 生产 | 审批、证据包、错误预算、值班机制 |

## 5. 回滚方式

MVP-practical 至少支持：

- 回滚 Agent manifest。
- 回滚 prompt。
- 禁用某个工具。
- 切回 mock/file adapter。
- 恢复上一版 policy。
- 恢复上一版 eval set 和 memory config。
- 禁用某个 MCP server 或切换到 curated allowlist。
- 停止新 run，保留旧 run 可查询。

## 6. 禁止发布

出现以下情况禁止发布：

- critical red team 未通过。
- audit event 缺失。
- trace 缺失。
- migration 不可回滚且没有备份。
- secret 出现在日志或 prompt。
- 写工具默认开启。
- 评测集没有更新但行为明显变化。
- checkpoint/resume 失败且没有恢复说明。
- destructive tool 缺少 operation_id、dry-run 或人工审批。
- sandbox policy 缺失但启用了代码、浏览器或 shell 类工具。
- 发布证据包无法锁定 prompt/model/tool/policy/eval set 组合。
