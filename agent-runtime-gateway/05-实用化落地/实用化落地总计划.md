# 实用化落地总计划

生成日期：2026-06-29  
目标：让本教学项目不止能讲清楚 Agent 系统，还能逐步落地成可实际应用的企业级 Agent Runtime/Gateway。

## 1. 实用化定义

本项目说“能实际应用”，不是指第一天就能进入生产环境，而是指最终能达到下面标准：

| 层级 | 含义 | 可接受状态 |
|---|---|---|
| 教学可运行 | 学生能在本地跑通核心链路 | mock model、mock tool、结构化测试 |
| 内部试点可用 | 小团队可用来处理低风险真实任务 | 只读工具、人工审批、完整审计 |
| 准生产可用 | 可接入企业内部系统试运行 | 身份、权限、评估、观测、恢复、隔离 |
| 生产可用 | 可承接真实业务流量 | SLO、告警、回滚、审计、合规、成本治理 |

当前目标不是直接跳到生产可用，而是按阶段从“教学可运行”推进到“内部试点可用”，并且从第一阶段开始采用工业级框架骨架。

## 1.1 工业级底线

本项目可以先小，但不能假。

即使是 MVP-practical，也必须具备：

- `Run/Step/State/Event` 显式模型。
- `trace_id` 全链路传递。
- `operation_id` 防重复副作用。
- `ErrorModel` 结构化错误。
- Tool Gateway 统一工具校验和授权。
- Audit Event 覆盖工具、策略、记忆写入。
- Checkpoint/Resume 设计。
- Golden/Red Team/Trajectory/Memory Eval。
- Release Gate 和 Rollback 计划。

没有这些，只能称为 demo，不能称为工业级框架。

## 2. 第一个实际应用场景

推荐场景保持不变：

**工单处理 Agent + 只读知识库 + 受控工具调用**

实际应用故事：

```text
员工提交一个 IT/客服/运营工单。
Agent 读取工单内容和只读知识库。
Agent 生成分类、优先级、建议回复和建议动作。
Agent 可调用只读工具查询状态。
涉及写操作时只生成待审批动作，不直接执行。
所有模型调用、工具调用、策略决策、记忆读写和人工审批都可追踪。
```

为什么适合真实落地：

- 业务价值清楚：减少一线处理重复工单的时间。
- 风险可控：先只读，不直接写生产系统。
- 可评估：有历史工单、标准答案、处理结果作为评测集。
- 可扩展：后续可加入审批、写操作、记忆、多 Agent、沙箱。

## 3. 实用化路线

| 阶段 | 教学目标 | 实用化目标 | 真实验收 |
|---|---|---|---|
| Phase 1 契约层 | 定义核心 schema | 所有运行输入输出可验证 | 坏输入被拒绝，错误可机器解析 |
| Phase 2 最小运行时 | 跑通 Agent loop | 本地可运行 demo | 一次 run 有 trace、step、结果和错误 |
| Phase 3 网关与工具治理 | 管住请求和工具 | 只读工具可安全调用 | 越权写操作被拒绝并审计 |
| Phase 4 长线任务 | 恢复和审批 | 工单任务可暂停/继续 | 中断恢复成功，重复操作不产生副作用 |
| Phase 5 MCP 工具生态 | 标准化接工具 | 替换 mock tool 为受控工具 | 工具注册、版本、权限、凭据隔离 |
| Phase 6 记忆系统 | 可控记忆 | 记住可验证偏好和历史处理模式 | 不写敏感信息，不跨租户召回 |
| Phase 7 测评审核 | 证明质量 | 每次变更有回归门禁 | golden/red team/memory/trajectory eval 通过 |
| Phase 8 可观测审计 | 复盘失败 | 运维能定位问题 | trace、audit、cost、latency 可查询 |
| Phase 9 安全隔离 | 隔离高风险工具 | 代码/浏览器工具进入沙箱 | 危险操作被拦截或隔离 |
| Phase 10 多 Agent | 协同处理复杂任务 | planner/reviewer/verifier 分工 | handoff 可审计，循环可拦截 |
| Phase 11 治理发布 | 上线和回滚 | 内部试点发布 | release gate、rollback、approval 可用 |

补充边界：

- **MVP-slice**：Phase 3 完成，证明 Runtime + Gateway + Tool + Policy + Audit。
- **MVP-practical**：Phase 4 完成，形成可部署内测版，具备 checkpoint、resume、审批、审计、基础评测。
- **MVP-complete**：Phase 9 完成，补齐 MCP、Memory、Eval、Observability、Sandbox。

## 4. 不做什么

为了让第一个版本真正能落地，先不做：

- 完整 no-code 工作流平台。
- 面向公众的 Agent marketplace。
- 真实生产写操作自动执行。
- 默认 GraphRAG。
- 复杂多云部署。
- 大规模多租户计费系统。
- 一开始就接代码执行和浏览器自动化。

这些不是永远不做，而是不能压进第一个可应用版本。

## 5. 实用化必备能力

要从教学项目变成实际应用，必须补齐：

- 数据模型：Run、Step、ToolCall、PolicyDecision、AuditEvent、MemoryRecord、EvalResult。
- 持久化：开发期 SQLite，后续 PostgreSQL。
- API：Request Gateway、Run API、Tool API、Eval API、Audit API。
- 认证：开发期 mock principal，试点期接 OIDC/IAM。
- 评估：golden set、red team set、memory set、trajectory set。
- 观测：trace_id、span、structured logs、metrics。
- 安全：tool allowlist、policy、secret isolation、sandbox profile。
- 运维：配置、环境变量、迁移、备份、告警、回滚。

## 6. 实用化判断问题

每个阶段完成后问：

```text
这个产物能否被下一阶段直接复用？
它失败时能否定位原因？
它能否阻止至少一种真实风险？
它是否有自动化验证？
它是否能被非作者看懂和维护？
```

五个问题都能回答，才算具备实际应用价值。
