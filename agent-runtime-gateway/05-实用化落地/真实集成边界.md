# 真实集成边界

生成日期：2026-06-29  
目标：明确什么时候用 mock，什么时候接真实系统，避免过早引入生产风险。

## 1. 三档 Adapter

| 档位 | 用途 | 示例 |
|---|---|---|
| mock adapter | 教学和单元测试 | 内存工单、内存知识库 |
| file/local adapter | 本地 demo 和评测 | JSON/CSV/Markdown 知识库 |
| real adapter | 内部试点和生产 | Jira、Zendesk、Confluence、ServiceNow、企业知识库 |

MVP-practical 只要求 mock + file/local。

## 2. 工单系统边界

第一版只做：

- 读取工单。
- 读取状态。
- 生成建议回复。
- 生成 proposed action。

第一版不做：

- 自动关闭工单。
- 自动改优先级。
- 自动发送邮件。
- 自动写生产 CRM/ITSM。

## 3. 知识库边界

第一版只接：

- 脱敏 Markdown。
- 本地 JSON。
- 小型示例 FAQ。

不接：

- 含敏感客户数据的真实知识库。
- 跨租户共享知识库。
- 未授权内部文档。

## 4. 真实系统接入门禁

接真实系统前必须满足：

- 有只读 API token。
- 有最小权限账号。
- 有审计日志。
- 有数据脱敏策略。
- 有评测集。
- 有回滚/关闭开关。
- 有用户告知和人工兜底。

## 5. Adapter 接口要求

所有 adapter 必须：

- 暴露 schema。
- 标注风险等级。
- 有超时。
- 有错误类型。
- 有审计摘要。
- 不直接把 secret 返回给 Agent。

## 6. 工具集成供应商风险

Composio、MCP server marketplace、第三方 automation 平台可以提高接入速度，但不能替代本项目自己的治理层。

采用任何外部工具集成前必须检查：

| 风险 | 必须回答 |
|---|---|
| 权限 | 它的 auth scope 是否能映射到本项目 principal/tenant/workspace？ |
| 审计 | 能否记录 tool_name、args_hash、result_hash、operation_id、trace_id？ |
| 凭据 | 凭据是否进入 prompt、日志、sandbox 或第三方不可控环境？ |
| 评测 | 是否能构造 mock、dry-run、redteam 和 regression case？ |
| 关闭 | 能否按工具、租户、Agent 维度禁用？ |
| 数据驻留 | 是否把企业数据传到外部云服务？ |
| 供应商锁定 | 替换供应商时 Tool Gateway 契约是否保持稳定？ |

结论：外部供应商最多是 adapter provider，不是安全边界、审计边界或发布门禁。
