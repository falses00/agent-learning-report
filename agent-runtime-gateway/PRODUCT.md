# Product

## Register

product

## Users

主要用户是想转向 Agent/LLM 应用工程的学习者。他听过 Agent、RAG、MCP、Memory 和 Eval，但缺少一条能持续写代码、复现故障、跑测试并形成作品集的路线。

用户不需要先成为框架专家，但必须愿意逐步完成 Python、HTTP、SQL、Git、pytest 和部署实验。界面应帮助他降低起步成本，不能替他跳过工程证据。

第二类用户是 AI 产品负责人、Agent 架构负责人和面试准备者。他们需要快速查找系统边界、故障模式、发布门禁和项目证据。

## Product Purpose

产品是一个静态的 Agent 工程学习控制台。它将课程文档、工程阶段、可运行基线、故障演练、自测和进度管理组合成一条明确工作流。

用户打开页面后应在一分钟内知道：

- 我是否需要先补工程基础。
- 当前应该学习哪个阶段。
- 这个阶段要实现什么、会发生什么事故、如何验证。
- 哪条命令可以立即运行。
- 通过后要留下什么作品集证据。

## Primary Journey

```text
选择起点
-> 运行教学基线
-> 进入当前阶段
-> 阅读最少必要材料
-> 实现工程增量
-> 跑正常/失败/对抗验证
-> 完成证据清单和自测
-> 标记阶段通过
-> 导出学习进度与作品集证据
```

## Core Functions

- 首次路径选择：工程基础补齐或直接进入 S0。
- 继续学习：记住当前阶段并显示唯一下一步。
- 阶段工作台：目标、交付物、故障、修复、证据和命令集中展示。
- 证据门禁：理解、实现、正常测试、失败测试、运行证据五项全部完成后才能标记通过。
- 故障演练：明确标注为前端教学模拟，不代表真实安全防护。
- 资料库：按开始、主线和参考分组，支持搜索和上一/下一篇。
- 本地进度：只写入浏览器 localStorage，可导出 JSON、可重置，不上传。

## Trust Boundaries

- 学习进度只保存在当前浏览器。
- 仿真不执行真实工具、不调用真实模型、不连接企业数据。
- 页面中的“通过”只代表本地学习记录，不等于生产认证或招聘保证。
- Markdown 内容来自本仓库，渲染时必须过滤危险标签和事件属性。
- 课程源码基线不能接入真实资金、客户数据或生产凭据。

## Success Criteria

- 首次用户无需滚动长报告即可进入一个明确任务。
- 用户能在两次点击内打开当前阶段或运行命令。
- 每个阶段都能看到事故、修复和验收证据。
- 只有五项证据完成后才能标记阶段通过。
- 桌面、平板和移动端都无内容重叠、横向溢出和不可达操作。
- 页面在外部图标或 Markdown CDN 失败时仍可完成核心导航与阅读。

## Language And Tone

中文优先，英文只保留工程术语。语气像资深工程导师：明确、克制、承认边界，不使用招聘保证或“生产已就绪”等误导文案。
