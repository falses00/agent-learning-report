# Phase 00.1 - 半懂起步教学手册

生成日期：2026-07-01  
学习模式：Design-only  
目标：在进入契约层之前，用一个工单退款故事建立工业级 Agent 的第一张直觉图。

## 1. 本阶段为什么存在

如果直接进入 Phase 1，学习者会马上遇到 `AgentManifest`、`ToolCall`、`PolicyDecisionInput`、`ErrorModel`。

这些概念都重要，但半懂学习者容易先记字段，后理解系统。Phase 0.1 的作用是先建立一个故事：

```text
客户申请退款。
Agent 可以建议，但不能直接执行。
执行必须经过身份、权限、工具、审批、审计、评测和发布门禁。
```

## 4. 本阶段要抓住的金句

```text
Agent 的建议不等于系统的执行许可。
```

如果你能理解这句话，后面的 Gateway、Policy、ToolCall、Audit、Eval 都会变得更自然。

## 5. Design-only 产物

| 产物 | 要求 |
|---|---|
| 一句话解释 | 工业级 Agent 和聊天 demo 的区别 |
| 工单故事复述 | 用退款工单讲出身份、权限、工具、审批、审计 |
| 坏设计卡 | 只靠 prompt 禁止退款为什么不够 |
| 复述卡 | 回答三个过关问题 |

## 6. 三个过关问题

1. 为什么“建议退款”和“执行退款”必须分开？
2. 如果工单处理到一半系统崩溃，只保存聊天记录有什么问题？
3. 如果新版本 Agent 退款判断变差，怎么提前发现？

## 7. 进入 Phase 1 条件

你能说出：

```text
工业级 Agent 不是聊天窗口。
它要能做长期任务，但所有执行都要受 Gateway、Policy、Audit、Eval 和 Release Gate 约束。
Phase 1 要做契约层，是为了让后续工具调用、权限判断、错误处理和审计都有结构化输入。
```

说不出，就不要进入 Phase 1。
