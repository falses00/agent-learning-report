## 1. 采用原则

小项目不按 stars 排名，而按“解决哪个具体瓶颈”采用。

| 采用方式 | 含义 |
|---|---|
| baseline | 可作为课堂最小可运行起点 |
| 专项实验 | 用来对比某个优化是否有效 |
| pattern only | 学思路，不引入依赖 |
| 待复核 | 需要复核许可证、维护状态、代码质量或复现条件 |

## 3. 教学采用策略

大项目用于基线：

- LangChain/LlamaIndex：解释生态和标准抽象。
- GraphRAG/LightRAG：解释图增强 RAG。
- Ragas/Promptfoo/Phoenix：解释评测和观测。

小项目用于手感：

- 每个实验只解决一个问题。
- 每个实验都要有 baseline、优化版、失败样例、指标对比。
- 小项目不直接进入生产架构，除非通过许可证、维护、安全和工程质量复核。

## 4. 风险边界

- 研究项目可能依赖旧模型、旧 API 或未锁版本。
- stars 很低不等于无价值，但必须先跑通最小实验。
- stars 很高也不等于安全可靠。
- 很多 demo 没有权限、审计、SLO、错误处理和多租户隔离。
- 本项目优先吸收 pattern，再重写成自己的工业级实现。
