# 参考项目采用准则

生成日期：2026-06-28  
用途：把 GitHub 调研结果转成可执行的采用规则，避免“看到热门项目就直接照搬”。

## 1. 三种采用方式

| 采用方式 | 含义 | 适用情况 | 例子 |
|---|---|---|---|
| `dependency allowed` | 可以作为候选依赖继续评估 | 许可证清楚、边界清楚、和核心架构不强绑定 | OpenTelemetry、OPA 等基础设施类项目 |
| `adapter candidate after license review` | 可做适配器候选，但必须先做许可证和商业条款复核 | 项目有价值，但许可证、托管方式或商业限制需要确认 | 部分 gateway、eval、observability 产品 |
| `pattern only` | 只学习设计模式，不直接引入依赖 | 项目太重、许可证不清、定位不同、或不应绑定生态 | OpenHands/Codex/Cline 的本地权限模型、MCP Registry 的发现模式 |

## 2. 不按 star 单独决策

GitHub star 只能说明关注度，不能说明：

- 生产可用性。
- 安全边界。
- 许可证适合商用。
- 企业部署成熟。
- API 稳定。
- 和本项目架构匹配。

所以每个项目都要同时看：

- 最近 push 和 release。
- open issues 和维护节奏。
- license 和商业条款。
- 是否为官方 spec、SDK 或 server。
- 能否拆成独立 adapter。
- 引入后是否锁死某个生态。

## 3. 采用前必须回答的问题

```text
这个项目解决本项目哪个模块的问题？
它是协议、基础设施、框架、产品，还是样例？
它是否必须成为依赖，还是只需要学习模式？
它的 license 是否清楚？
它是否会把核心架构绑定到某个生态？
它是否能被 adapter 包起来，未来替换？
如果不用它，我们是否仍能实现 MVP？
```

## 4. 本项目当前取舍

| 模块 | 推荐学习对象 | 采用方式 |
|---|---|---|
| Runtime | LangGraph、Temporal、AgentCore samples | 优先学习状态机、checkpoint、workflow 模式 |
| Tool Gateway | MCP SDK、GitHub MCP Server、FastMCP | MCP 兼容层可评估依赖，企业治理自建 |
| Model Gateway | LiteLLM、Portkey、Kong、Envoy | 学习路由、fallback、限流、审计模式 |
| Memory | LlamaIndex、RedisVL、GraphRAG | 分层引入，GraphRAG 不进 MVP |
| Evaluation | Promptfoo、Ragas、Langfuse、Phoenix | 组合离线评估和在线 trace/eval |
| Sandbox | gVisor、Firecracker、E2B、Wasmtime | 按风险分级采用，不把普通 Docker 当完整边界 |
| Coding Agent Lessons | OpenHands、Codex、Cline | pattern only，学习权限、diff、审批、workspace |

## 5. 2026-06-29 补充项目采用准则

| 问题域 | 参考项目 | 采用方式 | 教学用途 |
|---|---|---|---|
| 记忆系统 | Mem0、Zep、Graphiti、Letta、TencentDB-Agent-Memory、A-Mem、Hindsight | 多数 `pattern only` 或 `adapter candidate after license review` | 学习写入门禁、召回、遗忘、知识图谱、记忆评测 |
| 多智能体协同 | CAMEL、AG2、Langroid、MindSearch、AgentScope、AgentVerse | `pattern only` | 学习 handoff、共享状态、角色边界、协同 benchmark |
| 测评审核 | Promptfoo、Giskard、DeepEval、Opik、LangWatch、AgentOps | `pattern only` 起步，Promptfoo 可作为候选工具评估 | 学习 red team、trajectory eval、session replay、CI gate |
| 网关与工具治理 | Portkey、Plano、mcp-agent、mcp-use、Helicone | `pattern only` 或 adapter 候选 | 学习 model gateway、MCP lifecycle、observability proxy、guardrails |
| 安全与人工审批 | HumanLayer、Invariant、Garak | `pattern only` | 学习 human-in-the-loop、agent guardrails、LLM 漏洞扫描 |

补充原则：

- 专项项目优先进入“教学案例”和“评测对照”，不要过早进入核心依赖。
- 每个专项项目至少要回答：它解决哪个阶段的问题、能测什么、不能照搬什么。
- 记忆和多 Agent 项目变化很快，进入代码阶段前必须重新核验 release、license、API 和维护状态。

## 6. 教学判断句式

每次引入一个参考项目，都要能说清：

```text
我们参考 [项目]，是因为它在 [模块] 上提供了 [模式/能力]。
我们采用方式是 [dependency allowed / adapter candidate after license review / pattern only]。
我们不会照搬 [风险点]，因为本项目需要保留 [企业治理/安全/可替换性]。
```
