## 3. 核验结论

1. 本项目的主线不应从“多角色聊天”出发，而应从 `Gateway + Runtime + Tool Governance + Eval + Observability + Security` 出发。
2. LangGraph、Temporal、Microsoft Agent Framework 共同支持一个判断：长线 Agent 必须有 durable state、checkpoint、resume、worker/retry 和可观测运行态。
3. OpenAI Agents SDK、Microsoft Agent Framework 支持一个判断：多 Agent 协同必须有 handoff 契约、角色边界、guardrails、tracing，而不是自由对话。
4. Mem0、Letta 等记忆项目证明 memory 是独立系统，但企业场景必须补充权限、来源、TTL、删除、污染测试和 write gate。
5. Langfuse、Promptfoo 类项目证明评测和观测要闭环：CI 阻塞、线上 trace、失败样本回流 regression。
6. E2B、OpenHands、gVisor/Firecracker 思路说明沙箱需要按工具风险分层；Docker 不能被当作所有威胁的最终边界。
7. Composio 适合学习工具生态产品形态，但不能替代本项目自己的工具权限、审计、凭据和评测门禁。
