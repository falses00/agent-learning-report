# 前端 PRD 与教学一致性深度审查

日期：2026-07-02  
状态：v0.2 教学一致性修订记录  
范围：学习可视化前端 PRD、Open Design PRD、PRODUCT、DESIGN、课程地图、阶段验收清单  
结论：方向正确，但 v0.1 更偏 Runtime/Gateway/Trace/Eval，可视化对完整教学目标的显式覆盖不足；v0.2 已补齐覆盖矩阵、专项模块、高级感标准和 Codex 协作轨道。

## 1. 用户真实需求

用户不是要一个普通前端，也不是要一个生产控制台。真实需求是：

```text
不写代码
不记代码
通过中文优先、高级感、可交互的学习控制台
真正理解如何设计工业级 Agent + RAG
能判断长线运行、记忆、评测、安全、协同、网关、RAG 优化是否合格
能学会如何指挥 Codex 和 subagent 完成后续工程
```

## 2. v0.1 主要缺口

| 等级 | 缺口 | 影响 |
|---|---|---|
| P0 | 前端嵌入主线只显式覆盖 Phase 0.1/2/3/7/8/11 | Phase 4 长线任务、Phase 5 MCP、Phase 6 记忆、Phase 9 沙箱、Phase 10 多 Agent 容易被边缘化 |
| P0 | 没有把“每次先增强提示词、合理调用 subagent”做成可视化教学对象 | 用户学不到如何指挥 Codex，只是在看 Agent 系统图 |
| P0 | RAG 只有链路和评测，没有“问题 -> 诊断 -> 优化 -> eval”的交互结构 | 不能教会用户面对各种 RAG 问题如何判断和优化 |
| P1 | “高级感”主要是风格词，缺少可验收的产品工艺标准 | Product Design 后续可能生成漂亮但不教学的界面 |
| P1 | A/B/C 视觉方向原本像三选一，未明确组合产品结构 | 单一方向无法同时解决课程导航、系统边界和事故学习 |
| P1 | 记忆、沙箱、多 Agent 协同没有独立可视化模块 | 用户难以建立这些工业能力的系统图 |
| P2 | 版本状态仍是 v0.1 Draft，无法体现已进入教学一致性修订 | 后续维护者容易误判成熟度 |

## 3. v0.2 修订决策

### 3.1 产品结构

采用组合路线：

```text
C 学习路线地图作为首页和阶段导航
-> A 分层控制台作为 Runtime / Gateway / Policy / Eval 全链路页
-> B 事故推演工作台作为失败案例、阻塞判断和低分回流页
```

这个组合比单图路线更适合教学：

- C 解决位置感。
- A 解决系统边界。
- B 解决事故推演和验收证据。

### 3.2 完整教学覆盖

前端必须显式覆盖：

- Agent 网关与工具治理。
- 长线任务与断点恢复。
- RAG 问题诊断与优化。
- 记忆系统。
- 测评审核。
- 可观测性与审计。
- 安全隔离与沙箱。
- 多 Agent 协同。
- 治理发布。
- Codex 提示词增强与 subagent 协作。

### 3.3 高级感定义

高级感不是：

- 紫蓝渐变。
- 大圆角卡片墙。
- 营销 hero。
- 炫技动画。
- 假装生产就绪。

高级感是：

- 一眼知道当前 Phase。
- 一眼知道当前学习任务。
- 一眼知道事故、负责层和证据。
- 信息密度高但可扫描。
- 状态语义清楚，颜色、文案、图标一起表达。
- 低分能自然进入下一轮学习。

## 4. 修订后的硬验收

任一前端 PRD、线框、视觉方向或原型，必须满足：

- 每页显示：学习模式 Learning mode / 模拟数据 Mock data / 不执行真实操作 No real execution / 来源文件 Source files。
- 每页绑定 Phase、事故、负责层、source_files、复述题。
- 每个核心专题都有“坏设计会怎样”。
- RAG 页面必须按“问题 -> 诊断 -> 策略 -> 评测”展示。
- 记忆页面必须展示写入门禁、TTL、撤销、过期和审计。
- 沙箱页面必须展示工具风险等级、sandbox profile 和人工审批条件。
- 多 Agent 页面必须展示 planner/executor/reviewer/verifier 的输入、输出、停止条件和证据合并。
- Codex 协作轨道必须展示原始问题、增强提示词、推荐 subagent、期望证据和主线程整合。
- 低于 3 分的复述必须产生卡点标签和回流动作。
- 不允许真实发布、真实审批、真实退款、真实凭据或连接生产系统按钮。

## 5. 仍需警惕

当前文档已更接近用户真实目标，但还不能声称：

- 前端已经实现。
- 学习有效性已经被真实样本证明。
- Product Design 图可以直接当原型。
- 课程已经等于生产级工程框架。

后续必须通过 v0.3 低保真可点击原型和学习验证记录证明有效。

## 6. 下一步

进入 v0.3 前，先生成低保真交互任务清单：

1. C 首页：Phase 位置、当前任务、卡点、下一步。
2. A 全链路：节点详情、正常/失败路径、证据面板、复述卡。
3. B 事故推演：单事故聚焦、负责层、阻塞判断、低分回流。
4. 专项模块：RAG、记忆、沙箱、多 Agent、Codex 协作轨道。
5. 学习验证：看图前后复述评分、误解标签、回流对象。
