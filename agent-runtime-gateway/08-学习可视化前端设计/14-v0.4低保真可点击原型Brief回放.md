## 2. Brief 回放

```text
产品：工业级 Agent 学习可视化控制台。
目标用户：半懂学习者、未来 AI 产品/架构负责人、Codex 协作者。
核心目标：不用写代码，也能通过图、故事、失败路径、复述题和证据面板，理解工业级 Agent / RAG 的设计体系。
视觉路线：采用 C 学习路线地图 + A 分层控制台 + B 事故推演工作台的组合。
交互级别：低保真可点击原型，轻交互，全部 mock 数据。
语言策略：中文优先双语；中文负责理解，英文只作为工程术语锚点。
硬边界：学习模式 Learning mode / 模拟数据 Mock data / 不执行真实操作 No real execution / 来源文件 Source files 必须始终可见。
禁止事项：真实执行、真实发布、真实审批、真实退款、真实凭据、生产系统连接。
```

## 3. 视觉来源

Product Design 原型必须继承已有三张视觉方向图，而不是重新发明一套界面。

| 视觉来源 | 用途 | 原型定位 |
|---|---|---|
| [方向C-学习路线地图.png](视觉方向图/方向C-学习路线地图.png) | 低认知负担首页 | 作为首屏和学习导航骨架 |
| [方向A-分层控制台.png](视觉方向图/方向A-分层控制台.png) | 系统层与证据面板 | 作为全链路页骨架 |
| [方向B-事故推演工作台.png](视觉方向图/方向B-事故推演工作台.png) | 事故、坏设计、阻塞判断 | 作为失败学习页骨架 |

视觉策略：

- C 负责“我在哪、学什么、下一步是什么”。
- A 负责“系统如何受控运行”。
- B 负责“少了哪层会出事故”。
- RAG、LLM-Wiki、Trace/Eval/Release 必须使用同一套布局语言，不单独变成另一种产品。

## 4. 首轮原型范围

首轮最多做 6 个核心页面。

| 顺序 | 页面 | 目标 | 必须覆盖 |
|---|---|---|---|
| 1 | C 首页 / 学习路线地图 | 让用户 30 秒内知道当前 Phase、任务、卡点和下一步 | Phase、当前任务、复述分、低分回流、source_files |
| 2 | A 全链路分层控制台 | 解释 Agent Runtime / Gateway / Policy / RAG / Eval 的受控链路 | 正常链路、坏设计路径、证据面板、复述题 |
| 3 | B 事故推演工作台 | 从重复退款事故倒推负责层和阻塞点 | 事故链路、缺失层、阻塞判断、低分回流 |
| 4 | RAG 问题诊断器 | 解释 RAG 找不到、找错租户、引用不可信 | 三类 RAG 问题、指标、eval case、release blocking |
| 5 | LLM-Wiki 知识层 | 解释外部资料如何受审查进入课程 | ImportQueue、SourceReview、KnowledgeCard、KnowledgeVersion |
| 6 | 证据与门禁页 | 合并首轮 Trace / Audit / Eval / Release 学习模拟 | trace、audit、eval gate、release packet、rollback plan |

首轮不单独展开为完整页面的内容：

- 记忆生命周期：先放入 A 页或证据页的抽屉 / 卡片。
- 沙箱与工具风险：先放入 A 页的工具风险状态。
- 多 Agent 协同：先放入 Codex 协作轨道或底部学习回流。
- Codex 提示词增强：先放入 C 首页或证据页的学习辅助区。

这样做的原因：

```text
先把用户最容易误解的主链路做清楚，再逐步扩展专题页面。
```

## 5. 页面状态要求

每个核心页面都必须覆盖：

| 状态 | 原型表达 |
|---|---|
| default | 默认教学状态，显示当前 Phase、当前任务、mock 边界 |
| selected | 选中节点、事故或知识卡，右侧显示白话解释和证据 |
| bad_design | 显示坏设计路径和事故后果 |
| blocked | 显示为什么不能继续 |
| low_score | 显示缺失项、重讲任务和回流对象 |
| empty | 解释为什么为空，以及下一步如何产生记录 |
| error | 显示缺失 source_files、mock 对象或引用 |
| mobile_collapsed | 移动端改为 stepper / tabs / 折叠面板 |

## 6. 中文优先双语规则

| 区域 | 规则 |
|---|---|
| 导航 | 中文为主 |
| 按钮 | 中文动作 + 对象，例如“查看受控链路”“判断是否阻塞” |
| 架构节点 | 中文 + English canonical term，例如“运行时 Runtime” |
| 教学解释 | 中文白话优先 |
| schema / mock 字段 | 英文稳定字段 |
| 复述题 | 中文 |
| source_files | 保留真实文件路径，不翻译 |

英文不能抢主视觉。

## 7. 不做什么

本阶段不做：

- Next.js 工程。
- 真实登录。
- 真实数据库。
- 真实工具调用。
- 真实生产发布。
- 真实审批。
- workflow builder。
- 大而全的企业后台。
- 营销 hero。

如果任何页面让用户以为可以真实执行或发布，本轮原型失败。

## 8. Brief 确认标准

Product Design brief gate 要求用户显式确认。仅有项目文档整理完成，不等于已经授权进入 ideate / prototype。

进入 Product Design 低保真可点击原型前，必须确认以下 8 点：

1. 首轮原型只做 6 个核心页面。
2. 视觉来源采用 A/B/C 组合，不重新换风格。
3. 所有页面都显示学习边界提示。
4. 所有数据都是 mock。
5. 所有按钮都是中文学习动作，不是真实操作。
6. 每页都有事故、负责层、证据、复述题和低分回流。
7. RAG 三类问题必须同时出现。
8. 原型完成后按 13 个 P0 验证主题做学习验证。
