# v0.4-pre 原型前冻结包

日期：2026-07-02  
状态：Product Design 原型前冻结包  
阶段判断：从 v0.3 页面流进入 v0.4-pre，不进入代码实现  
目标用户：不写代码、不记代码、通过可视化和复述学会工业级 Agent/RAG 设计的学习者

## 1. 本阶段结论

下一步不应该直接写 Next.js，也不应该马上做可运行前端。

最合理的下一步是：

```text
v0.3 低保真页面流
-> v0.4-pre 原型前冻结包
-> Product Design 低保真可点击原型
-> 学习验证
-> 再决定是否进入静态前端实现
```

原因：

- 当前项目的目标是让用户学会设计、评审和验收，不是先把界面跑起来。
- 已有 PRD、v0.3 页面流和 LLM-Wiki 知识层，但逐页 mock 数据、状态矩阵、学习验证脚本还没有冻结。
- 如果现在直接写前端，最大风险是做成漂亮 dashboard，却不能证明用户真的理解事故、负责层和证据。

## 2. Product Design brief 播放稿

```text
产品：工业级 Agent 学习可视化控制台。
对象：半懂学习者、未来 AI 产品/架构负责人、Codex 协作者。
目标：通过中文优先的可视化，让用户看懂 Agent Runtime/Gateway/RAG 的受控长线链路。
路线：C 学习路线地图作为首页，A 分层控制台作为全链路页，B 事故推演工作台作为失败学习页。
新增重点：LLM-Wiki 知识层，展示外部资料如何经 SourceReview、KnowledgeCard、KnowledgeVersion 和 ImportQueue 进入课程。
交互：低保真到轻交互，只用 mock 数据，不接真实后端。
边界：学习模式 Learning mode / 模拟数据 Mock data / 不执行真实操作 No real execution / 来源文件 Source files 必须始终可见。
禁止：营销 hero、真实发布、真实审批、真实退款、真实凭据、连接生产系统。
```

Product Design 下一步只能基于这份 brief 进入低保真原型或视觉探索；不能绕过 brief 直接生成代码原型。

## 5. 半懂学习者要在这一步解决的卡点

| 卡点 | 原型必须怎么帮 |
|---|---|
| 以为 Agent 是聊天窗口 | C 首页和 A 链路必须展示受控系统，而不是聊天框 |
| 以为 ToolCall 就是执行 | A 链路必须展示 ToolCall 是申请，Policy 决定 allow/deny/approval |
| 以为 RAG 检索到就可信 | RAG 诊断器必须展示引用、freshness、tenant ACL 和 eval case |
| 以为记忆越多越好 | 记忆生命周期必须展示写入门禁、TTL、撤销和审计 |
| 混淆 trace 和 audit | Trace 页必须双栏对照定位与证明 |
| 把 eval 当分数 | Eval 页必须展示 release blocking |
| 以为多 Agent 越多越强 | 多 Agent 页必须展示角色边界、停止条件和证据合并 |
| 以为 GitHub 热门项目可以直接采用 | LLM-Wiki 页必须展示 SourceReview、license、review_status 和版本 |
| 复述低分后不知道怎么办 | 底部回流条必须展示卡点、重讲任务、下次复习和回流对象 |

## 6. Product Design 原型前硬门禁

进入 Product Design 低保真可点击原型前，必须全部满足：

- 页面状态矩阵覆盖默认、选中、阻塞、低分、空态、错误、移动端折叠。
- mock 数据字典覆盖 Phase、FailureCase、LayerCard、RAGDiagnostic、EvalCase、Trace/Audit、ReleasePacket、RestatementRecord、LearningBacklog、EnhancedPrompt、SubagentTask、KnowledgeSource、SourceReview、KnowledgeCard、KnowledgeVersion、ImportQueue。
- 每个页面都有 `source_files`。
- 每个页面都有复述题。
- 每个页面都有低分回流路径。
- 所有按钮是中文动作，不出现真实执行、真实发布、真实审批。
- 所有核心状态不只靠颜色表达。
- 用户如果把原型误认为生产系统，原型判失败。

## 8. 真实建议

本阶段真正要冻结的不是“界面长什么样”，而是：

```text
用户每点击一页，是否更能说清一个工业级判断。
```

如果页面不能让用户说出事故、负责层和证据，再高级的视觉也不能进入下一步。
