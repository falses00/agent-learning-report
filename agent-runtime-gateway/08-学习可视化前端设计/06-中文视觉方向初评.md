## 4. 推荐原型路线

不建议只选一张图作为唯一方向。更适合本教学项目的路线是：

```text
C 学习路线地图作为首页
-> A 分层控制台作为全链路学习页
-> B 事故推演工作台作为失败案例页
-> 复述评分和低分回流贯穿三页
```

原因：

- 用户需要先知道自己在哪个阶段，C 的主入口价值最高。
- 用户真正理解工业级 Agent，需要看清 Runtime / Gateway / Policy / Eval 的边界，A 的系统图价值最高。
- 用户要经得住测评审核，需要能从事故倒推拦截层和证据，B 的失败推演价值最高。

## 6. 高级感风险清单

进入原型前必须修正这些风险：

- 视觉图里的文字只作方向参考，不能直接切图使用。
- 所有中文文案必须在真实前端中重写，避免生成图文字失真。
- A 方向要补 RAG、记忆、沙箱、多 Agent 的入口，否则系统边界不完整。
- B 方向要控制事故密度，避免初学者被失败列表压住。
- C 方向必须保持“路线 + 系统链路 + 事故 + 回流”，不能退化成目录。
- 所有图都必须补 Codex 协作轨道：原始问题、增强提示词、推荐 subagent、期望证据、主线程整合。

## 7. 进入前端原型前的硬门禁

- 原型必须用真实 HTML/CSS/React 文本重建，不依赖图片里的文字。
- 每页必须显示：学习模式 Learning mode / 模拟数据 Mock data / 不执行真实操作 No real execution / 来源文件 Source files。
- 主导航、按钮、复述题必须中文。
- 关键工程词采用“中文 + English canonical term”。
- 低分复述必须生成卡点标签、重讲任务、RAG 评测样例或下次复习。
- 不允许出现真实发布、真实审批、真实退款、真实凭据或连接生产系统按钮。

## 8. 下一步建议

下一步应进入 v0.3 低保真可点击原型设计：

1. 先用 C 做首页信息架构。
2. 点击当前阶段后进入 A 的全链路页面。
3. 点击事故或低分卡点后进入 B 的事故推演页面。
4. 所有页面共用同一套 mock 学习记录、Phase 数据、失败案例、RAG eval case 和 source_files。
