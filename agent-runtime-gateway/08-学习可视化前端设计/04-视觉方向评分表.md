# 视觉方向评分表

状态：Product Design ideate 前置评分标准  
用途：生成 3 个视觉方向后，用同一把尺子决定哪个方向进入原型

## 1. 三个候选方向

### 方向 A：分层控制台

核心想法：

把教学过程呈现为一个受控系统控制台。左侧是阶段，中心是“运行时 Runtime / 请求网关 Request Gateway / 工具网关 Tool Gateway”链路图，右侧是证据面板。

适合：

- 教 Phase 2、3、8、11。
- 强调系统边界、trace、audit、release gate。

风险：

- 容易太像真实生产控制台。
- 必须显著显示“学习模式 Learning mode”和“模拟数据 Mock data”。

### 方向 B：事故推演工作台

核心想法：

把失败案例放在中心，让用户先选事故，再看到负责层、拦截点和复述题。

适合：

- 教 Phase 3、4、6、7、9、10。
- 强调坏设计如何变成事故。

风险：

- 如果事故列表太密，会压迫半懂学习者。
- 必须保持每次只解释一个核心失败。

### 方向 C：学习路线地图

核心想法：

把 Phase 0.1-11 做成渐进路线图，用户每阶段打开一张学习卡，查看设计产物、坏设计、复述评分和下一阶段门禁。

适合：

- 首屏和主入口。
- 帮助用户知道自己现在在哪一步。

风险：

- 容易变成课程目录。
- 必须和链路图、事故、trace/eval 结合，不能只做导航。

## 2. 评分维度

每项 0-5 分，低于 3 分不能进入原型。

| 维度 | 5 分标准 | 0 分标准 |
|---|---|---|
| 学习清晰度 | 一眼知道当前学什么、为什么学、下一步做什么 | 像普通 dashboard，不知道学习任务 |
| 工业边界 | 清楚显示运行时 Runtime、请求网关 Request Gateway、工具网关 Tool Gateway、策略 Policy、评估门禁 Eval Gate 等边界 | 把 Agent 画成一个聊天框 |
| 失败路径 | 坏设计和事故链路明显 | 只有 happy path |
| 复述验收 | 复述卡和评分门禁自然可见 | 复述被藏在角落或没有 |
| Mock 边界 | 学习模式 Learning mode、模拟数据 Mock data、不执行真实操作 No real execution 明显 | 容易误认为真实生产系统 |
| 信息密度 | 桌面端可扫描，移动端可折叠 | 信息堆积或空洞 |
| 可访问性 | 颜色、文字、图标共同表达状态 | 关键状态只靠颜色 |
| 可扩展性 | 后续能接 mock JSON、trace、eval、release packet | 只能做静态展示 |
| 中文化 | 中文主文案清楚，英文术语只做辅助 | 主要界面全英文或中英混乱 |
| 学习迭代 | 能看到复述低分如何回流到下一轮学习 | 只展示一次性课程内容 |

## 3. Product Design 生成提示词骨架

用于生成视觉方向时的共同约束：

```text
Create a realistic, production-quality desktop product UI design, 1440 x 1024, for an industrial Agent learning visualization console.

Language strategy: Chinese-first bilingual UI. Main navigation, buttons, teaching questions, evidence labels, restatement cards, and phase labels must be in Simplified Chinese. English appears only as small helper labels for canonical technical terms, such as 运行时 Runtime, 请求网关 Request Gateway, 模型网关 Model Gateway, 工具网关 Tool Gateway, 策略 Policy, 检索增强 RAG, 轨迹 Trace, 审计 Audit, 评估门禁 Eval Gate, 发布门禁 Release Gate.

The user is a semi-technical Chinese learner who does not want to write code. The UI teaches Agent Runtime/Gateway/RAG design through a refund-ticket story, controlled system flow, failure simulation, trace replay, eval gate, and release learning simulation.

This is a learning console, not a production operations console and not a marketing website. Do not show real publish, real approval, real refund execution, real credentials, or production connection buttons.

Every screen must visibly include Chinese-first boundary badges: 学习模式 Learning mode, 模拟数据 Mock data, 不执行真实操作 No real execution, 来源文件 Source files.

Style: quiet, trustworthy, scannable, product UI, industrial learning console. Avoid marketing hero, decorative card walls, purple-blue gradients, huge rounded cards, fake production controls, and real publish buttons.

Use a restrained OKLCH-inspired palette with neutral surfaces, readable ink, semantic accent, warning, danger, blocked, and success states.

Use app shell structure: top status bar, left phase rail, main visual workspace, right evidence panel, bottom reflection strip.

Show sustainable learning iteration: 复述评分, 卡点标签, 下次复习, and low-score feedback loop from answer to 术语解释 / 失败案例 / RAG 评测样例 / 复述卡.
```

## 4. 三方向差异化要求

方向 A 提示补充：

```text
Emphasize the layered runtime architecture. Main workspace is a clickable Chinese-first system flow. Right panel explains selected layer, accident prevented, source files, and restatement question.

Chinese-first UI labels must read: 用户请求 User Request -> 请求网关 Request Gateway -> 运行时 Runtime -> 工具网关 Tool Gateway -> 策略 Policy -> 工具 Tools -> 轨迹/审计 Trace/Audit -> 评估/发布门禁 Eval/Release Gate.

The right panel must show: 白话解释、拦截事故、负责层、来源文件、复述题、复述评分、卡点标签、下次复习。
```

方向 B 提示补充：

```text
Emphasize failure-driven learning. Main workspace starts from failure cases like duplicate refund, unauthorized tool, memory pollution, missing trace, eval downgrade. Show accident chain, responsible layer, blocking decision, and restatement card.

Chinese-first UI labels must read: 重复退款、越权工具、记忆污染、缺失轨迹、评测降级、事故链路、应拦截层、阻塞判断、复述卡、低分回流动作。

Show only one primary failure case in focus, with the rest as compact side options.
```

方向 C 提示补充：

```text
Emphasize phase progression. Main workspace shows Phase 0.1-11 as a learning route, with current phase open. Each phase card contains goal, accident, design artifact checklist, bad design example, restatement score, and next gate.

Chinese-first UI labels must read: 学习目标、事故样例、设计产物清单、坏设计、复述评分、下一阶段门禁、下次复习。

Do not make it a plain course directory. Connect the open phase to 链路图、事故、Trace/Eval、低分回流。
```

## 5. 进入原型条件

必须同时满足：

- 用户选择 A/B/C 中一个方向。
- 该方向评分总分 >=32，且单项没有低于 3。
- 用户确认仍然是 mock learning console。
- Product Design brief 已确认。
- 不接真实后端、真实凭据或真实生产写操作。
