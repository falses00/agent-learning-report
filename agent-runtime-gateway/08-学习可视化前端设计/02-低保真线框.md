# 低保真线框

状态：Open Design 线框稿  
目标：先定义布局和学习任务，不定义最终视觉细节

## 1. App Shell

```text
┌──────────────────────────────────────────────────────────────┐
│ 顶部状态：学习模式 | 模拟数据 | 不执行真实操作 | 来源文件       │
├───────────────┬───────────────────────────────┬──────────────┤
│ 阶段栏         │ 主视觉工作区                    │ 证据面板      │
│               │                               │              │
│ Phase 0.1     │ 当前链路图/时间线/表格           │ 事故          │
│ Phase 1       │                               │ 负责层        │
│ Phase 2       │                               │ 证据          │
│ ...           │                               │ 来源文件      │
├───────────────┴───────────────────────────────┴──────────────┤
│ 复述栏：复述评分 | 下一步 | 停止原因 | 下次复习            │
└──────────────────────────────────────────────────────────────┘
```

## 2. 学习首页

布局：

```text
顶部状态：学习模式 Learning mode / 模拟数据 Mock data / 不执行真实操作 No real execution / 来源文件 Source files
Phase 0.1 入口
退款工单故事
三个今日问题
主操作：查看受控链路
右侧证据面板：今天不做什么
复述栏：记录复述评分 / 加入下次复习
```

验收：

- 30 秒内能指出当前 Phase。
- 能说出主动作。
- 能说出不接真实系统。

## 3. 全链路可视化页

布局：

```text
Left: phase rail
中心：用户请求 User Request -> 请求网关 Request Gateway -> 运行时 Runtime -> 工具网关 Tool Gateway -> 策略 Policy -> 工具 Tools -> 轨迹/审计 Trace/Audit -> 评估/发布门禁 Eval/Release Gate
右侧：选中节点解释
底部：正常路径 / 坏设计路径切换
```

节点点击后显示：

- 白话解释。
- 工单故事映射。
- 防什么事故。
- 少了会怎样。
- 对应 Phase。
- 复述问题。

## 4. Phase 学习页

布局：

```text
阶段标题
一句话目标
事故优先
系统边界
Design-only 产物清单
坏设计案例
复述卡 Restatement
下一阶段门禁
```

硬规则：

- 一页只教一个核心判断。
- 不用大段长文。
- 每页必须有坏设计。

## 5. 失败推演页

布局：

```text
失败案例列表 | 事故链路 | 应拦截层
```

交互：

- 选择坏设计。
- 显示事故链路。
- 标出应该拦截的层。
- 生成复述题。

## 6. Trace 回放页

布局：

```text
步骤时间线 | span/event 详情 | 轨迹 Trace vs 审计 Audit 对照
```

必须表达：

- trace 用于定位。
- audit 用于证明。
- secret 不进入两者。

## 7. Eval 门禁页

布局：

```text
版本选择
case 表
失败原因
门禁判断
必须重跑的评测
```

必须表达：

- 不只测最终答案。
- trajectory/tool/policy/memory 都要测。
- critical failure 阻塞发布。

## 8. Release 学习模拟页

布局：

```text
变更摘要
release packet 版本组合
eval 重跑矩阵
模拟门禁判断
rollback 学习样例
```

按钮只允许：

- 查看版本组合。
- 模拟发布判断。
- 查看 rollback 学习样例。

禁止：

- 发布。
- 执行生产发布。
- 连接真实系统。
