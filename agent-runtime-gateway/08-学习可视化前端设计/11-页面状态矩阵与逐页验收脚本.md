# 页面状态矩阵与逐页验收脚本

日期：2026-07-02  
状态：v0.4-pre 原型前资产  
用途：让 Product Design 原型和后续前端实现有统一的页面状态、验收脚本和停止条件

## 1. 全局状态矩阵

所有页面必须至少覆盖以下状态：

| 状态 | 含义 | 必须显示 |
|---|---|---|
| default | 默认教学状态 | 当前 Phase、当前任务、mock 边界 |
| selected | 选中节点或案例 | 白话解释、事故、负责层、证据 |
| bad_design | 坏设计路径 | 事故链路、应拦截层、阻塞原因 |
| blocked | 门禁阻塞 | 阻塞类型、证据、不能继续的原因 |
| low_score | 复述低分 | 缺失项、重讲任务、回流对象 |
| empty | 暂无记录 | 为什么为空、下一步如何产生记录 |
| error | mock 数据错误或引用缺失 | 缺失 source_files / 数据对象 |
| mobile_collapsed | 移动端折叠 | 主任务优先，证据面板下移为 tabs |

全局固定文案：

```text
学习模式 Learning mode
模拟数据 Mock data
不执行真实操作 No real execution
来源文件 Source files
```

## 2. 页面级状态矩阵

| 页面 | default | selected | blocked / low_score | empty / error | 移动端 |
|---|---|---|---|---|---|
| C 首页 | 当前 Phase、今日任务、主行动 | 展开当前 Phase 卡 | 低分时主按钮变为“回看卡点” | 无学习记录时显示第一课入口 | Phase rail 改为顶部 segmented control |
| A 全链路 | 正常受控链路 | 选中 Gateway / Runtime / Policy / RAG / Eval | 显示绕过 Tool Gateway 或 Eval blocked | 缺 source_files 时显示修复提示 | 链路改为纵向 stepper |
| B 事故推演 | 默认“重复退款”事故 | 选中坏设计 | 阻塞判断解释 pass / review / blocked | 无事故时显示默认事故 | 事故列表折叠为 chips |
| RAG 诊断器 | 默认“引用不可信” | 选中问题标签 | 跨租户、胡编、引用不可信可阻塞 | 缺 eval case 时提示补评测 | 指标表横向滚动 |
| 记忆生命周期 | 候选记忆到 audit event | 选中写入门禁 / TTL / 撤销 | 记忆污染触发 blocked | 无候选记忆时显示示例 | 泳道改为 stepper |
| 沙箱与工具风险 | 工具风险矩阵 | 选中工具类型 | 高风险工具显示 approval / sandbox | 未知工具类型需 SourceReview | 矩阵改为列表 |
| 多 Agent 协同 | planner -> executor -> reviewer -> verifier | 选中角色 | 冲突写入显示 resource lock 缺失 | 无 handoff 时显示单 Agent 边界 | handoff 改为纵向 |
| Codex 协作 | 原始问题 -> 增强提示词 -> subagent | 选中 subagent 角色 | 缺期望证据时阻止进入执行 | 无外部资料时不推荐 researcher | 面板下移 |
| LLM-Wiki 知识层 | ImportQueue -> SourceReview -> KnowledgeCard | 选中来源 / 知识卡 / 版本 | blocked_by_license / deprecated | 缺 license 或 source_files 时错误 | 主题树折叠 |
| Trace 回放 | mock run 时间线 | 选中 span/event | 缺 audit event 时提示无法证明 | 无 trace 时显示样例 run | 时间线纵向 |
| Eval 门禁 | case 矩阵 | 选中 case | critical 失败 release blocked | 缺 case 时不能发布模拟 | 表格折叠为卡片 |
| Release 学习模拟 | release packet | 选中版本项 | eval 降级或证据缺失 blocked | 无 rollback plan 时不能继续 | checklist 纵向 |

## 3. 逐页验收脚本

每个页面都用同一套 9 问验收：

| 问题 | 通过标准 |
|---|---|
| 当前 Phase 是否可见？ | 30 秒内能指出 |
| 当前任务是否明确？ | 主行动只有一个 |
| 这页防什么事故？ | 能说出具体事故 |
| 负责层是谁？ | 能落到 Gateway / Runtime / Policy / RAG / Memory / Eval / Observability / Sandbox / Governance |
| 最小证据是什么？ | 有 source_files、trace、eval case、audit event 或 mock 数据 |
| 是否有坏设计路径？ | 能看到少了哪层会怎样 |
| 是否有复述题？ | 题目要求事故 + 负责层 + 证据 |
| 低分如何回流？ | 有 Concept / FailureCase / EvalCase / RestatementCard / KnowledgeCard 目标 |
| 是否可能误读为生产系统？ | 若是，直接不通过 |

## 4. 页面验收样例

### 4.1 C 首页

验收动作：

1. 打开首页。
2. 指出当前 Phase、今日任务和主按钮。
3. 说出页面为什么不是生产控制台。
4. 点击“查看受控链路”。

过关复述：

```text
我现在在 Phase 0.1。
今天学的是建议和执行分离。
如果模型建议直接变成退款执行，会造成越权退款。
负责层是 Gateway / Policy。
证据是页面显示学习模式、模拟数据、不执行真实操作。
```

### 4.2 A 全链路页

验收动作：

1. 点击 Tool Gateway。
2. 切换坏设计路径。
3. 指出绕过 Policy 会发生什么。
4. 提交复述。

过关复述：

```text
ToolCall 是工具申请，不是执行。
如果绕过 Tool Gateway 和 Policy，模型建议可能直接触发退款。
负责层是 Tool Gateway 和 Policy。
证据是 ToolCall 申请、PolicyDecision 和 audit event。
```

### 4.3 LLM-Wiki 知识层页

验收动作：

1. 打开 ImportQueue。
2. 选择一个 GitHub 来源。
3. 查看 SourceReview 和 KnowledgeCard。
4. 判断是否能进入课程。

过关复述：

```text
这条知识来自 GitHub 来源。
它不能因为 star 高就直接进入课程。
负责层是 SourceReview 和 KnowledgeVersion。
证据是 license_status、review_status、source_files 和 adoption_mode。
低分会回流到 ImportQueue 或 KnowledgeCard。
```

## 5. 停止条件

出现以下任一情况，停止进入 Product Design 原型：

- 任一核心页没有 mock 边界提示。
- 任一核心页没有坏设计路径。
- 任一核心页没有复述题。
- 低分只显示分数，没有回流对象。
- LLM-Wiki 页把外部资料当作可信正文。
- Release 页出现真实发布按钮。
- RAG 页让用户以为“检索命中 = 可信事实”。
