# v0.4 低分回流修订清单

日期：2026-07-02  
用途：当学习者评分低于 3 时，不继续推进，而是把问题回流到具体内容资产。

## 1. 回流原则

低分不是失败，是课程修订信号。

但低分不能只记录分数，必须绑定：

```text
缺失项 -> 回流对象 -> 修订动作 -> 再验证问题
```

## 2. 回流对象说明

| 回流对象 | 什么时候用 | 修订动作 |
|---|---|---|
| Concept | 名词听不懂 | 用退款工单重讲 |
| FailureCase | 不知道为什么危险 | 补坏设计事故 |
| EvalCase | 不知道怎么验收 | 补评测样例和阻塞条件 |
| RestatementCard | 说不出自己的话 | 改填空题和追问 |
| StateMatrix | 页面状态不够清楚 | 补 blocked / low_score 状态说明 |
| SourceReview | 外部资料可信度误解 | 补来源、license、freshness 审核 |
| KnowledgeCard | 知识进入课程路径不清 | 补知识卡字段和版本 |
| PolicyCase | 权限边界不清 | 补 allow / deny / audit 样例 |
| TraceCase | 事故定位不清 | 补 trace_id、span、audit event |
| RAGCase | RAG 问题类型判断不清 | 补找不到、找错租户、引用不可信样例 |
| CitationCase | 引用可信度判断不清 | 补 citation、freshness、source_files 判断 |
| MemoryGate | 记忆写入边界不清 | 补来源、权限、TTL、撤销和审计 |
| AuditCase | 不知道什么需要审计 | 补主体、策略、工具、结果和错误字段 |
| ReleaseGate | 不知道何时阻塞发布 | 补 release blocking 和回滚条件 |
| BlockerRule | 不知道普通低分和阻塞失败区别 | 补安全、跨租户、red team 阻塞规则 |
| SandboxProfile | 不知道工具如何隔离 | 补 no sandbox、restricted、approval 的选择理由 |
| ToolRiskCase | 不知道工具风险等级 | 补只读、写操作、高风险外发工具样例 |
| ApprovalRule | 不知道何时人工审批 | 补高风险动作的审批条件和审计字段 |
| HandoffCase | 多 Agent 交接不清 | 补任务、状态、证据、剩余风险和停止条件 |
| ConflictCase | 多 Agent 冲突不清 | 补资源锁、仲裁、停止条件 |
| ReviewerPrompt | 不知道 reviewer 如何审 | 补发现坏设计、缺证据、越界风险的审查提示词 |
| ReplayQuestion | 不知道如何复盘 trace | 补从事故回放到具体层的追问题 |

## 3. 主题到回流对象

| 主题 | 低分常见原因 | 优先回流 |
|---|---|---|
| 建议与执行分离 | 把建议当执行许可 | Concept / FailureCase / RestatementCard |
| Runtime 状态 | 把聊天记录当状态 | Concept / StateMatrix / FailureCase |
| Tool Gateway | 以为 prompt 能管权限 | PolicyCase / EvalCase / FailureCase |
| 长线任务 | 不懂 retry 和副作用 | FailureCase / StateMatrix / EvalCase |
| RAG 找不到 | 只想加 top_k | RAGCase / EvalCase / RestatementCard |
| RAG 找错租户 | 不知道这是安全阻塞 | PolicyCase / EvalCase / FailureCase |
| RAG 引用可信 | 认为检索命中就可信 | SourceReview / CitationCase / EvalCase |
| 记忆污染 | 认为记越多越好 | FailureCase / AuditCase / MemoryGate |
| Eval 门禁 | 认为 eval 是分数装饰 | EvalCase / ReleaseGate / BlockerRule |
| Trace/Audit | 混淆调试日志和审计证据 | TraceCase / AuditCase / ReplayQuestion |
| 沙箱风险 | 认为 Docker 足够安全 | SandboxProfile / ToolRiskCase / ApprovalRule |
| 多 Agent 协同 | 认为多数投票即可 | HandoffCase / ConflictCase / ReviewerPrompt |
| LLM-Wiki 知识层 | 认为热门资料可直接入课 | SourceReview / KnowledgeCard / KnowledgeVersion |

## 5. 低分后的提示词

```text
我刚才这个主题低于 3 分。
请不要进入下一页。
请根据我的缺失项，只重讲一个概念。
要求用退款工单故事说明：事故、负责层、最小证据。
最后只问我一个复述题，并重新评分。
```
