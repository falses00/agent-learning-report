# v0.4 学习效果验证计划

日期：2026-07-02  
对象：`v0.4-低保真可点击原型`  
目标：证明原型是否真的帮助半懂学习者理解工业级 Agent/RAG 设计，而不是只证明页面能打开。

## 1. 验证原则

本验证只看学习效果，不看代码能力。

每个主题都要验证三件事：

```text
事故
负责层
最小证据
```

如果学习者无法同时说出这三件事，即使页面好看、交互正常，也不能算通过。

## 2. 验证范围

必须覆盖 13 个 P0 主题：

| 编号 | 主题 | 原型页面 | 必须识别的事故 | 负责层 | 最小证据 |
|---:|---|---|---|---|---|
| 1 | 建议与执行分离 | C 首页 / A 全链路 | 越权退款 | Request Gateway / Policy | 学习边界、ToolCall 申请、PolicyDecision |
| 2 | Runtime 状态 | A 全链路 / 证据页 | 任务中断后无法恢复 | Runtime | Run / Step / Checkpoint |
| 3 | Tool Gateway | A 全链路 | 模型直接拿危险工具 | Tool Gateway / Policy | 工具申请、策略结果、AuditEvent |
| 4 | 长线任务 | B 事故 | retry 重复副作用 | Runtime / Checkpoint | operation_id、resume、blocked |
| 5 | RAG 找不到 | RAG 诊断器 | 缺关键政策仍胡答 | Retrieval / Eval | hard retrieval case、query rewrite、recall |
| 6 | RAG 找错租户 | RAG 诊断器 | 跨租户引用合同 | Policy / RAG / Tool Gateway | tenant ACL、blocked case |
| 7 | RAG 引用可信 | RAG 诊断器 | 引用未审核来源 | Citation / SourceReview / Eval | citation、freshness、source_files |
| 8 | 记忆污染 | 证据页 / 后续 Phase 6 | 错误事实长期召回 | Memory Gate / Audit | TTL、撤销、来源 |
| 9 | Eval 门禁 | 证据页 | red team 失败仍发布 | Eval / Release Gate | release blocking、eval case |
| 10 | Trace/Audit | 证据页 | 事故后无法证明过程 | Trace / Audit | trace_id、audit event |
| 11 | 沙箱风险 | 证据页 / 后续 Phase 9 | 高风险工具外发数据 | Sandbox / Approval | 工具风险等级、approval |
| 12 | 多 Agent 协同 | 证据页 / 后续 Phase 10 | 多 agent 冲突或投票误判 | Supervisor / Reviewer / Verifier | handoff、resource lock、证据合并 |
| 13 | LLM-Wiki 知识层 | LLM-Wiki 页 | 未审核资料进入课程 | SourceReview / KnowledgeVersion | license_status、review_status、KnowledgeCard |

## 3. 每个主题的验证动作

每个主题按同一流程：

1. 看图前提问。
2. 记录 `before_score`。
3. 打开对应页面或状态。
4. 切换一次坏设计或低分回流状态。
5. 让学习者再次回答。
6. 记录 `after_score`。
7. 追加一个坏设计判断题。
8. 标记缺失项：事故、负责层、证据、边界、回流。
9. 判断 Go / Conditional Go / No-Go。

## 4. 评分规则

| 分数 | 判断 | 是否过关 |
|---:|---|---|
| 0 | 把 mock 当生产，或完全说不出 | 不过 |
| 1 | 只背术语 | 不过 |
| 2 | 能说大意，但缺负责层或证据 | 不过 |
| 3 | 能说出事故、负责层、最小证据 | 过关下限 |
| 4 | 能指出坏设计和停止条件 | 通过 |
| 5 | 能主动提出评测、审计、回归和实现门禁 | 优秀 |

## 5. Go / Conditional Go / No-Go

| 判定 | 条件 | 下一步 |
|---|---|---|
| Go | 至少 11/13 个 P0 主题 >=3，且 RAG 三类全部 >=3，无生产误读 | 进入 Phase 1 契约层 |
| Conditional Go | 只有 1-2 个非 RAG 主题低于 3，且都有回流对象 | 先补低分项，再进入 Phase 1 |
| No-Go | RAG 任一类低于 3，或出现生产误读，或低分无回流 | 停止推进，重讲对应主题 |

## 6. 验证时禁止

- 禁止要求学习者写代码。
- 禁止把原型称为生产控制台。
- 禁止把“会点击”当作“学会了”。
- 禁止跳过低分回流。
- 禁止用 star、检索命中或 LLM 总结代替 SourceReview。
