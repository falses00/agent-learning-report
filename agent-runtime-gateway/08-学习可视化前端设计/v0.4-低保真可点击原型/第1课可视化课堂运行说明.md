# 第1课可视化课堂运行说明

日期：2026-07-02  
对象：半懂学习者  
模式：Design-only，不写代码，不记代码  
配套原型：`index.html`

## 1. 本课目标

这节课只解决一个核心直觉：

```text
Agent 的建议不等于系统的执行许可。
```

你要学会的不是前端，也不是框架，而是看到一个 Agent/RAG 页面时，能判断：

1. 它防什么事故。
2. 哪一层负责拦截或记录。
3. 最小证据是什么。
4. 低分时应该回到哪里重学。
5. 为什么这不是生产控制台。

## 2. 本轮增强提示词

学习时直接把下面这段发给 Codex：

```text
开始第 1 课：用 v0.4 可视化原型理解工业级 Agent。

我半懂不懂，不想写代码，也不想记代码。
请先调用 planner 或 verifier subagent 帮我增强本轮学习提示词。
本轮只使用 v0.4 低保真可点击原型和第一节课文档，不进入工程实现。

教学要求：
1. 先带我打开原型首页，确认这是学习模式、模拟数据、不执行真实操作。
2. 每一页都按“事故 -> 负责层 -> 最小证据 -> 复述题 -> 低分回流”讲。
3. 不要堆术语，每个术语都用退款工单故事解释。
4. 我回答后再评分，低于 3 分不要进入下一页。
5. 每次评分都记录到学习效果评分表或学习者复述评分记录。
6. 最后给出 Go / Conditional Go / No-Go 判断。
```

## 3. 课堂顺序

| 步骤 | 打开位置 | 要看什么 | 必须回答 |
|---|---|---|---|
| 1 | 首页 | 学习模式、模拟数据、不执行真实操作 | 为什么这不是生产控制台？ |
| 2 | C 首页 | 学习路线地图 | 为什么“建议”和“执行”要分开？ |
| 3 | A 全链路 | Gateway / Policy / Tool / Runtime | ToolCall 为什么只是申请？ |
| 4 | B 事故 | 重复退款、越权执行、缺 checkpoint | retry 为什么可能造成重复副作用？ |
| 5 | RAG 诊断器 | 找不到、找错租户、引用不可信 | 为什么检索命中不等于可信答案？ |
| 6 | LLM-Wiki | ImportQueue / SourceReview / KnowledgeCard / KnowledgeVersion | 为什么 GitHub 热门资料不能直接进课程？ |
| 7 | 证据与门禁 | source_files、状态覆盖、复述评分 | 什么情况下不能进入下一阶段？ |

第 7 步不是只看一页结束，而是必须逐行补齐 13 个 P0 主题评分。尤其不要漏掉这些容易被页面流带过的主题：

| 必查主题 | 最小追问 | 低于 3 分回流 |
|---|---|---|
| 记忆污染 | 长期记忆为什么不能默认写入？ | MemoryGate / AuditCase |
| Eval 门禁 | 哪些失败必须阻塞下一阶段？ | EvalCase / ReleaseGate |
| Trace/Audit | trace 用来定位什么，audit 用来证明什么？ | TraceCase / AuditCase / ReplayQuestion |
| 沙箱风险 | 高风险工具为什么需要隔离或审批？ | SandboxProfile / ToolRiskCase / ApprovalRule |
| 多 Agent 协同 | 多 Agent 分歧为什么按证据合并？ | HandoffCase / ConflictCase / ReviewerPrompt |

课堂结束前必须打开 [学习效果评分表](学习效果评分表.md)，确认 13 行都有 before / after / 缺失项 / 回流对象 / 判定。

## 4. 每页固定讲法

每一页只按这个顺序讲：

```text
现实事故
-> 系统负责层
-> 最小证据
-> 坏设计会怎样
-> 复述评分
-> 低分回流
```

不要先讲框架名，不要先讲代码结构。

## 5. 学习者必须完成的复述

每一页结束前，你必须能用自己的话说：

```text
这一页防的是 ________ 事故。
负责层是 ________。
最小证据是 ________。
如果少了这一层，会发生 ________。
如果我复述低于 3 分，应该回到 ________。
```

## 6. 本课停止条件

出现任一情况，停止继续讲新内容：

- 你把页面理解成真实生产控制台。
- 你只会背 Gateway、Runtime、RAG，但说不出事故。
- 你说得出事故，但说不出负责层。
- 你说得出负责层，但说不出最小证据。
- 你把 RAG 理解成“搜到了就可信”。
- 你把 LLM-Wiki 理解成“外部资料可以自动进入课程”。

## 7. 通过标准

本课不要求满分。最低通过标准是：

```text
13 个 P0 主题至少 11 个看图后评分 >=3。
RAG 三类问题全部评分 >=3。
没有生产系统误读。
所有低分项都有回流对象。
```

通过后才进入 Phase 1 契约层。
