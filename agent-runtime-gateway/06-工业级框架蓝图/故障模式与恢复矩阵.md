# 故障模式与恢复矩阵

生成日期：2026-06-30  
目标：提前定义每层怎么失败、怎么检测、怎么恢复。

| 层 | 故障模式 | 检测 | 恢复 | 是否阻塞发布 |
|---|---|---|---|---|
| Request Gateway | 未认证请求进入 runtime | auth test、audit | 拒绝请求、记录事件 | 是 |
| Runtime | run 卡死 | step timeout、max steps | 标记 recoverable、checkpoint resume | 是 |
| Runtime | checkpoint 损坏 | resume test、checksum | 回退上一 checkpoint 或 terminal failure | 是 |
| Model Gateway | provider 超时 | model span timeout | fallback、retry、降级 | 视影响 |
| Model Gateway | 成本超预算 | cost meter | 停止 run、切低成本模型 | 是 |
| Tool Gateway | schema 漏校验 | bad-case tests | 拒绝调用、修 schema | 是 |
| Tool Gateway | 重复副作用 | operation ledger | 返回已有结果、阻止重复执行 | 是 |
| Policy | 越权写放行 | red team、policy test | 禁用工具、修策略 | 是 |
| Memory | 错误事实写入 | memory eval | 标记失效、加入纠错记录 | 是 |
| Memory | 跨租户召回 | privacy eval | 停用 memory retrieval、修过滤 | 是 |
| MCP | server 不健康 | health check | 熔断、禁用 server | 视影响 |
| Sandbox | 危险操作逃逸 | adversarial probe | 停用 provider、升级隔离 | 是 |
| Observability | trace 缺失 | coverage check | 阻塞发布、修 instrumentation | 是 |
| Audit | audit event 缺失 | audit coverage | 阻塞发布、补审计 | 是 |
| Evaluation | eval runner 失效 | CI failure | 冻结发布 | 是 |
| Storage | DB 迁移失败 | migration check | 回滚迁移、恢复备份 | 是 |

## 恢复原则

- 能安全恢复的，进入 `failed_recoverable`。
- 不能安全恢复的，进入 `failed_terminal`。
- 所有恢复动作必须保留 audit event。
- 所有故障必须能通过 trace_id/run_id 关联。
- 修复后的故障必须进入 regression set。
