## 1. 采用原则

外部项目不是答案库，而是风险提醒器。一个项目 star 很高，只能说明值得看；不能证明它适合成为本项目依赖。

本项目采用三层判断：

| 判断 | 含义 |
|---|---|
| 架构基线 | 可影响本项目核心抽象和验收标准 |
| 模式参考 | 借鉴设计模式或产品形态，不绑定依赖 |
| 历史启发 | 用来理解行业演进和反例，不作为新主线 |

## 3. 不照搬规则

- 不照搬黑盒多角色聊天，把多 Agent 协同变成可审计 handoff。
- 不照搬自动记忆写入，把 memory write gate 交给规则、来源、权限和评测。
- 不照搬公共 MCP server 信任，把所有 server 纳入准入审核。
- 不照搬工具集成供应商的权限模型，本项目必须自有 Tool Gateway。
- 不照搬单一 eval 平台，本项目底层事件和数据集要可迁移。

## 4. 教学使用方式

每个阶段讲外部项目时，必须用同一格式：

```text
这个项目解决什么问题：

本项目不采用什么：

```

如果无法把外部项目经验落到本项目的 schema、状态机、门禁、评测、运行指标或教学练习，就不要把它写入主路线。
