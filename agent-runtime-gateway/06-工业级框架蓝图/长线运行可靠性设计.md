# 长线运行可靠性设计

生成日期：2026-06-29  
目标：让 Agent 能执行长线任务，而不是只能完成短对话。

## 1. 长线任务风险

长线 Agent 最大风险不是一次回答错，而是：

- 多步执行后状态丢失。
- 工具失败后重复副作用。
- 人工审批后无法接上。
- 上下文越来越大且越来越脏。
- 模型换 provider 后行为漂移。
- 任务循环、重复、卡死。
- 评测只看最终答案，漏掉过程错误。

## 2. 核心对象

| 对象 | 用途 |
|---|---|
| `Run` | 一次长线任务 |
| `Step` | 一次可审计动作 |
| `State` | 当前结构化状态 |
| `Event` | 模型、工具、策略、记忆、人工动作 |
| `Checkpoint` | 可恢复快照 |
| `OperationId` | 防重复副作用 |
| `TraceId` | 跨层复盘 |
| `ApprovalTicket` | 人工审批状态 |

## 3. Workflow / Activity / Worker 模型

长线 Agent 任务可以借鉴 durable execution 的分层，但不能把模型自由循环直接当成工作流引擎。

| 概念 | 本项目解释 | 可靠性要求 |
|---|---|---|
| Workflow | 一个 Run 的高层流程和状态机 | 可重放、可恢复、状态迁移可审计 |
| Activity | 一个 Step 内的外部动作，如模型调用、工具调用、记忆写入 | 可重试、有 timeout、有结构化错误 |
| Worker | 执行 Run/Step 的运行单元 | 有心跳、可抢占、可恢复 |
| Retry Policy | 针对 Activity 的重试规则 | 最大次数、backoff、终止条件 |
| Resume Policy | 针对 Run 的恢复规则 | 从可信 checkpoint 恢复，不重复副作用 |
| Visibility | 运行态查询能力 | 可按 run_id、trace_id、agent_version、status 查询 |

教学上要区分：

- Workflow 保证流程和恢复语义。
- Agent 决定下一步意图。
- Gateway 决定是否允许工具执行。
- Worker 只是执行，不拥有安全决策权。

## 4. 状态机要求

最小状态：

```text
created
-> running
-> waiting_tool
-> waiting_approval
-> checkpointed
-> completed
-> failed_recoverable
-> failed_terminal
-> cancelled
```

要求：

- 每个状态迁移必须有原因。
- 每个失败必须分类。
- recoverable 失败必须给出恢复入口。
- terminal 失败必须给出审计证据。
- waiting_approval 必须能恢复。

## 5. Checkpoint 设计

checkpoint 至少包含：

- run_id。
- agent_version。
- state_version。
- current_step。
- task_goal。
- accumulated_facts。
- pending_tool_calls。
- pending_approvals。
- memory_refs。
- trace_id。
- created_at。

不能只保存 chat history。

## 6. 幂等与重试

任何可能产生副作用的操作必须有：

- `operation_id`。
- 幂等键存储。
- 重复请求检测。
- retry policy。
- 最大重试次数。
- backoff。
- terminal failure。

示例：

```text
同一个 operation_id 再次提交写工单动作：
如果已成功 -> 返回已完成结果。
如果执行中 -> 返回 pending。
如果失败可恢复 -> 允许 retry。
如果失败不可恢复 -> 返回 terminal error。
```

## 7. 循环防护

必须实现或计划：

- 最大 step 数。
- 最大工具调用数。
- 最大连续失败数。
- 重复状态检测。
- 重复工具参数检测。
- 无进展检测。
- 人工升级。

## 8. 上下文治理

长线运行不能无限堆上下文。

必须区分：

- 当前任务目标。
- 可靠事实。
- 待验证事实。
- 工具结果。
- 人工审批结果。
- 可丢弃历史。
- 禁止写入记忆的内容。

Context packer 必须基于重要性、来源、权限和预算，而不是只按时间截断。

## 9. 长线运行验收

至少测试：

- 正常多步任务完成。
- 模型调用超时后恢复。
- 工具失败后恢复。
- 进程中断后 resume。
- 重复 `operation_id` 不重复执行。
- 人工审批后继续。
- 最大 step 触发停止。
- trace 能复盘完整过程。

## 10. 工业级指标

| 指标 | 目标 |
|---|---:|
| checkpoint_resume_success | >= 99% |
| duplicate_side_effect_rate | 0 critical cases |
| trace_coverage | 100% run |
| audit_coverage | 100% tool/policy/memory write |
| loop_guard_effective | 100% known loop cases |
| recoverable_error_classification | >= 95% |
