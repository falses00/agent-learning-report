# 工具风险分级与审批矩阵

生成日期：2026-06-30  
目标：定义不同工具风险对应的默认策略、审批和隔离要求。

| 风险级别 | 工具类型 | 示例 | 默认策略 | 必需控制 |
|---|---|---|---|---|
| L0 | 纯本地计算 | 格式化、摘要、分类 | 允许 | trace |
| L1 | 只读低敏数据 | FAQ 查询、脱敏知识库 | 允许 | schema、audit |
| L2 | 只读内部数据 | 工单查询、内部状态查询 | 条件允许 | principal、tenant、policy、audit |
| L3 | 写操作 dry-run | 生成待审批动作 | 允许但不执行 | approval ticket、audit |
| L4 | 真实写操作 | 改工单、发邮件、写数据库 | 默认审批 | policy、HITL、operation_id、audit |
| L5 | 代码/浏览器/外网 | shell、browser、HTTP 外发 | MVP 禁止 | sandbox、egress policy、审批 |
| L6 | 生产凭据/跨租户 | 管理员 token、客户数据 | 默认禁止 | 特批、强审计、隔离 |

## 审批规则

- L0-L1：自动执行，但必须 trace。
- L2：根据 principal/resource/action 判断。
- L3：只生成 proposed action。
- L4：需要 approval ticket。
- L5：需要 sandbox profile + approval。
- L6：默认禁止，必须人工特批。

## Sandbox Profile 分层

| Profile | 适用工具 | 网络 | 文件系统 | 凭据 | 默认结论 |
|---|---|---|---|---|---|
| none | L0-L1 纯本地/只读低敏 | 禁止外发 | 无写权限 | 不注入 | 允许 |
| restricted-process | L2 只读内部查询 | allowlist | 只读挂载 | scoped token | 条件允许 |
| docker-sandbox | L5 文件/命令实验 | 默认禁网 | 临时目录 | 禁止生产凭据 | 需审批 |
| gvisor/microvm | 不可信代码、高风险自动化 | 强 allowlist | 隔离文件系统 | 禁止生产凭据 | 高风险候选 |
| cloud-sandbox | 浏览器/代码执行外包环境 | egress policy | 临时 workspace | 短期凭据或无凭据 | 需数据驻留审查 |
| no-production-secret | 所有 profile 的底线 | 不适用 | 不适用 | 生产 secret 禁止进入 prompt/sandbox/log | 强制 |

Sandbox 不能替代 Tool Gateway。即使工具在沙箱内执行，也必须有 schema、policy、operation_id 和 audit。

## Tool Registry 必填字段

- `tool_name`
- `tool_version`
- `owner`
- `risk_level`
- `schema`
- `allowed_roles`
- `allowed_operations`
- `requires_approval`
- `requires_sandbox`
- `credential_scope`
- `audit_fields`
- `timeout_seconds`
- `idempotency_required`

## 阻塞条件

- 工具没有 risk_level。
- 写工具没有 operation_id。
- 高风险工具没有 sandbox profile。
- 工具结果没有过滤策略。
- 工具调用没有 audit event。
