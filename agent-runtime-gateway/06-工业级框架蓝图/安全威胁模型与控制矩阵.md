# 安全威胁模型与控制矩阵

生成日期：2026-06-30  
目标：让 Agent 安全不是靠 prompt，而是靠威胁模型、控制措施和验证。

## 1. Agent 专项威胁

| 威胁 | 攻击方式 | 控制措施 | 验证 |
|---|---|---|---|
| Prompt Injection | 用户诱导模型忽略规则 | 工具策略独立于 prompt、red team | 注入用例必须拒绝 |
| Tool Injection | 工具返回恶意指令 | 工具结果标记为 untrusted、结果过滤 | 工具输出诱导不得执行 |
| 越权访问 | 伪造 principal/resource | RBAC/ABAC、tenant/workspace 过滤 | 跨租户用例必须拒绝 |
| Secret 泄漏 | 模型读取 env/log/tool result | Credential Broker、redaction | secret scanner、日志检查 |
| 数据外传 | 调外部 URL 或写工具 | egress policy、tool allowlist | 外发用例必须阻塞 |
| 记忆污染 | 写入错误/敏感/恶意记忆 | memory write gate、来源、TTL | memory red team |
| 重复副作用 | retry 导致重复写 | operation_id、ledger | duplicate test |
| Sandbox Escape | 高风险工具逃逸 | gVisor/microVM/E2B profile | adversarial probe |
| 审计绕过 | 绕过 Gateway 调工具 | 所有工具只经 Tool Gateway | audit coverage 100% |

## 2. 控制层次

```text
Schema
-> Policy
-> Credential Broker
-> Result Filter
-> Audit
-> Sandbox
-> Eval/Red Team
```

任何一层失败，不能依赖 prompt 补救。

## 3. Sandbox 与凭据边界

| 场景 | 最低隔离 | 凭据规则 | 阻断条件 |
|---|---|---|---|
| 只读知识库查询 | Tool Gateway + result filter | scoped read token | 返回 secret 或跨租户结果 |
| 本地文件处理 | restricted workspace | 不注入生产凭据 | 读取 workspace 外路径 |
| shell/代码执行 | Docker/gVisor/microVM profile | 禁止生产凭据 | 网络未控、文件系统未隔离 |
| 浏览器自动化 | cloud/local sandbox + egress allowlist | 短期最小权限 | 访问非 allowlist 域名 |
| 外部写操作 | dry-run + approval + operation_id | Credential Broker 注入 | 无审批或无幂等键 |

安全隔离必须和风险分级绑定，不能由模型临时决定。

## 4. 安全门禁

禁止进入 prod-lite：

- secret 出现在 prompt、trace、audit、eval report。
- critical 注入用例可绕过工具策略。
- 跨租户资源可被读取。
- 写操作没有 approval/dry-run。
- 高风险工具没有 sandbox profile。
- audit coverage 低于 100%。
