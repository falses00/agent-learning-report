# 测评审核升级蓝图

生成日期：2026-06-30  
目标：把评测从“跑用例”升级为可审计、可阻塞发布的审核系统。

## 1. 评测分层

| 层 | 用途 | 工具形态 |
|---|---|---|
| rule assertions | schema、安全、权限、secret | deterministic tests |
| golden eval | 常规业务正确性 | fixtures + scorer |
| red team eval | 注入、越权、泄漏 | adversarial cases |
| trajectory eval | 过程和工具选择 | trace assertions |
| memory eval | 召回、污染、删除、隔离 | memory fixtures |
| online audit | 线上抽样和人工复核 | dashboard/report |

## 2. 阈值矩阵

| 指标 | 阈值 | 阻塞 |
|---|---:|---|
| critical security block rate | 100% | 是 |
| schema validation pass | >= 99% | 是 |
| tool call accuracy | >= 95% | 是 |
| trace coverage | 100% | 是 |
| audit coverage | 100% | 是 |
| memory leakage critical | 0 | 是 |
| false refusal | <= 2% | 视场景 |
| cost per run | 预算内 | 视场景 |

## 3. Release Audit Packet

每次发布生成：

```text
release_id
agent_version
prompt_version
model_version
tool_registry_version
policy_version
memory_policy_version
eval_set_version
test_command
passed_cases
failed_cases
critical_failures
trace_samples
audit_samples
approval_record
rollback_plan
```

## 4. 线上回流

prod-lite 后必须：

- 抽样人工复核。
- 事故样本进入 regression。
- 用户纠错进入评测集候选。
- 新型注入进入 red team。
- 失败 trace 进入 trajectory eval。

## 4.1 CI + Online + Regression 闭环

| 环节 | 输入 | 输出 | 阻塞规则 |
|---|---|---|---|
| CI gate | prompt/model/tool/policy/runtime diff | eval report、trace samples | critical fail 直接阻塞 |
| Online audit | prod-lite trace、audit、metrics | drift report、incident candidate | P0/P1 触发冻结 |
| Regression intake | 事故、用户纠错、红队新样本 | regression case、owner、due date | 未入库不得关闭事故 |
| Release review | CI + online + regression 证据 | Release Audit Packet | 证据不全禁止 beta 以上发布 |

## 5. LLM-as-judge 限制

LLM-as-judge 只能辅助评估主观质量，不能用于：

- 权限判断。
- secret 泄漏判断。
- schema 合法性。
- 跨租户访问。
- critical 安全放行。
