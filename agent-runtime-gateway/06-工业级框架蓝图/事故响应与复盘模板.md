# 事故响应与复盘模板

生成日期：2026-06-30  
目标：让事故能变成永久防线，而不是只修一次。

## 1. 事故等级

| 等级 | 示例 | 响应 |
|---|---|---|
| P0 | secret 泄漏、越权写执行、跨租户泄漏 | 立即停用、通知、复盘 |
| P1 | audit 缺失、评测门禁失效、checkpoint 大面积失败 | 冻结发布、修复 |
| P2 | 模型超时、工具失败率升高、成本超预算 | 降级、限流 |
| P3 | 文档或非阻塞测试缺口 | 排期修复 |

## 2. 事故响应流程

```text
检测
-> 分级
-> 限制影响
-> 保全 trace/audit
-> 修复或回滚
-> 复测
-> 复盘
-> 新增 regression
-> 更新门禁
```

## 4. 永久防线

事故结束必须至少产生一个：

- regression case。
- red team case。
- policy rule。
- alert rule。
- runbook 更新。
- release gate 更新。

没有永久防线，事故不算关闭。
