# 生产就绪审核清单

生成日期：2026-06-30  
目标：定义从 prod-lite 进入 production 前必须通过的最终审核。

## 1. 架构审核

- [ ] 分层契约清晰。
- [ ] Runtime 不绕过 Tool Gateway。
- [ ] Tool Gateway 不依赖 prompt 做权限。
- [ ] Memory 有写入门禁和召回权限。
- [ ] Eval 有阻塞发布能力。
- [ ] Observability 覆盖全链路。

## 2. 可靠性审核

- [ ] Run/Step 状态机完整。
- [ ] checkpoint 恢复测试通过。
- [ ] operation_id 防重复副作用。
- [ ] prod-lite 阶段：超时、重试、熔断、DLQ 可以是明确计划，但必须写清风险、临时控制、负责人和截止日期。
- [ ] production 阶段：超时、重试、熔断、DLQ 必须已经实现，并通过故障注入或恢复演练证明有效。
- [ ] 备份恢复演练通过。

production 不接受“有计划但未实现”的关键可靠性能力。计划只能作为 prod-lite 风险登记，不能作为生产放行证据。

## 3. 安全审核

- [ ] secret 不进入 prompt/log/trace。
- [ ] critical red team 100% 阻塞。
- [ ] 高风险工具有 sandbox profile。
- [ ] 跨租户访问测试通过。
- [ ] 工具风险分级和审批矩阵已应用。

## 4. 测评审核

- [ ] smoke/golden/red-team/regression 可一键运行。
- [ ] trajectory eval 覆盖工具选择和恢复。
- [ ] memory eval 覆盖污染、删除、泄漏。
- [ ] release audit packet 完整。
- [ ] 失败 case 已归档。

## 5. 运维审核

- [ ] SLO 已定义。
- [ ] 告警已配置。
- [ ] runbook 已存在。
- [ ] 事故复盘模板已存在。
- [ ] 回滚路径已演练。
- [ ] 成本预算和容量上限已配置。

## 6. 治理审核

- [ ] Agent/prompt/model/tool/policy/eval 都有版本。
- [ ] 发布需要审批。
- [ ] 回滚可定位到具体版本组合。
- [ ] audit export 可用。
- [ ] 数据保留和删除策略明确。

## 7. 最终结论

只有所有 critical 项通过，才允许 production。

如有 P1/P2 残留风险，必须写入：

```text
风险：
影响：
临时控制：
负责人：
截止日期：
是否允许上线：
批准人：
```
