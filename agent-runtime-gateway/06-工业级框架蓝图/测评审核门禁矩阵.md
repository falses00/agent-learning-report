# 测评审核门禁矩阵

生成日期：2026-06-29  
目标：让系统经得住测评审核，而不是只靠主观感觉。

## 1. 门禁分层

本矩阵描述工业级工程门禁。当前 Design-only 学习阶段先产出等价设计证据；进入 Implementation-later 后，再把这些证据转成自动化测试、真实 trace、eval report 和发布审批。

| 门禁 | 工程触发时机 | Design-only 等价证据 | 阻塞级别 |
|---|---|---|---|
| Schema Gate | 每次代码提交 | 契约字段责任卡、坏输入表 | 阻塞 |
| Unit Gate | 每次代码提交 | 单模块行为清单、边界样例 | 阻塞 |
| Integration Gate | 每次合并 | 跨层流程图、接口责任表 | 阻塞 |
| Golden Eval Gate | prompt/model/tool/policy 变更 | golden case 设计和预期行为 | 阻塞 |
| Red Team Gate | 发布前 | 注入、越权、泄漏失败样例 | 阻塞 |
| Memory Eval Gate | memory policy 或 retrieval 变更 | 记忆污染、过期、跨租户样例 | 阻塞 |
| Trajectory Gate | runtime/orchestration/tool 变更 | 期望步骤、禁止步骤、恢复路径 | 阻塞 |
| Observability Gate | 发布前 | span/audit 字段设计和复盘路径 | 阻塞 |
| Security Gate | 发布前 | 威胁模型、风险分级、阻塞策略 | 阻塞 |
| Cost Gate | 试点前 | 成本预算、top_k/rerank/cache 策略 | 可阻塞 |

## 2. 评测维度

| 维度 | 必测内容 | 失败后果 |
|---|---|---|
| 答案质量 | 分类、建议回复、证据引用 | 阻塞发布或降级 |
| 工具选择 | 是否调用正确工具 | 阻塞发布 |
| 工具参数 | schema、类型、权限资源 | 阻塞发布 |
| 轨迹质量 | 重复调用、无进展、错误恢复 | 阻塞发布 |
| 记忆质量 | 写入、召回、遗忘、污染、泄漏 | 阻塞发布 |
| 安全红队 | 注入、越权、secret、外传 | critical 必须阻塞 |
| Gateway | 路由、预算、fallback、限流 | 阻塞或降级 |
| 可观测 | trace、audit、replay 覆盖 | 阻塞试点 |
| 多 Agent | handoff、角色边界、仲裁 | 阻塞多 Agent 阶段 |

## 3. 最低评测集结构

```text
22-评测集/
  smoke/
  golden/
  red-team/
  memory/
  trajectory/
  gateway-policy/
  sandbox/
  multi-agent/
```

每个 case 至少包含：

- `case_id`
- `phase`
- `input`
- `expected_behavior`
- `forbidden_behavior`
- `expected_tools`
- `expected_policy`
- `assertions`
- `risk_level`

## 4. 审核报告必须包含

```text
run_id
trace_id
agent_version
prompt_version
model_version
tool_registry_version
policy_version
memory_snapshot
eval_set_version
passed_cases
failed_cases
critical_failures
cost_summary
latency_summary
release_decision
```

## 5. 不能通过审核的情况

出现任一情况，禁止发布或进入下一阶段：

- critical red team case 未拦截。
- tool call 无 audit event。
- policy decision 无记录。
- memory write 无来源。
- trace 不完整。
- prompt injection 绕过工具策略。
- 重复 operation_id 产生重复副作用。
- 评测失败但没有加入 regression set。

## 6. 人工审核要求

自动评测之外，还要人工审核：

- 高风险策略变更。
- 新增真实工具。
- 新增长期记忆写入规则。
- 新增外部系统 adapter。
- 发布到内部试点。

人工审核必须留下：

- 审核人。
- 时间。
- 依据。
- 通过/拒绝。
- 剩余风险。

## 7. 工业级最低目标

| 指标 | 阶段目标 |
|---|---:|
| schema pass rate | >= 99% |
| tool call accuracy | >= 95% |
| critical security block rate | 100% |
| trace coverage | 100% |
| audit coverage | 100% tool/policy/memory write |
| checkpoint resume success | >= 99% |
| memory leakage critical cases | 0 |
| duplicate side effect critical cases | 0 |
